# 界面设计规范

本文档定义系统所有界面的设计规范，包括全局布局、导航结构、页面模板和各模块页面的详细要求。视觉风格参考现有 Accountability 项目（Tailwind CSS + 深色侧边栏 + 卡片式布局），适配中国企业会计准则（CAS）的业务场景。

**页面总览：** 9 个模块、41 个主要页面/视图、5 种标准模板

---

## 第 1 节：全局布局规范

### 1.1 AppShell 三区结构

```
┌──────────────────────────────────────────────────────────────────┐
│  TOPBAR                                                          │
│  [组织名称]  [公司选择器 ▼]  [当前期间状态]          [用户头像] │
├──────────────┬───────────────────────────────────────────────────┤
│  SIDEBAR     │  MAIN CONTENT                                     │
│  240px 固定  │  [面包屑路径]                                     │
│  slate-900   │  ┌─────────────────────────────────────────────┐  │
│  不随内容滚动 │  │ 页面标题                      [主操作按钮]  │  │
│              │  ├─────────────────────────────────────────────┤  │
│  [导航菜单]  │  │                                              │  │
│              │  │  页面主体内容区域（可滚动）                  │  │
│              │  │                                              │  │
│  ─────────── │  └─────────────────────────────────────────────┘  │
│  [系统设置]  │                                                   │
└──────────────┴───────────────────────────────────────────────────┘
```

### 1.2 顶部导航栏（Topbar）

| 位置 | 内容 | 说明 |
|------|------|------|
| 左侧 | 组织名称 | 纯文字标识，不可点击 |
| 中左 | 公司选择器 | 下拉菜单，列出当前组织下所有有效公司；切换后全局数据刷新 |
| 中右 | 当前期间状态 | 格式："2025年3月 · 开放"；开放=绿色文字，关闭=红色文字 |
| 右侧 | 用户头像 | 点击展开：个人设置 / 退出登录 |

**设计要求：**
- 背景：white，底部有 1px border-b（slate-200）
- 高度：56px（固定）
- 公司选择器：选择框宽度自适应公司名称，最大宽度 240px

### 1.3 侧边栏（Sidebar）

| 属性 | 值 |
|------|-----|
| 宽度 | 240px，固定不可拖拽 |
| 背景色 | slate-900 |
| 位置 | 左侧，Topbar 以下，全高 |
| 滚动 | 内容区滚动，侧边栏不动 |
| 菜单文字（普通） | text-slate-400 |
| 菜单文字（激活） | text-white |
| 激活样式 | 左侧 3px 蓝色竖条（blue-500）+ 背景 slate-800 |
| 折叠行为 | 一级菜单可展开/折叠；默认展开当前模块，其余折叠 |
| 系统设置位置 | 侧边栏底部，固定（不随菜单滚动） |

### 1.4 主内容区（Main Content）

- 背景：gray-50（浅灰，与侧边栏形成对比）
- 内边距：px-8 py-6（内容与边缘的间距）
- 最大宽度：无限制（全宽自适应）
- 面包屑：位于页面标题上方，文字小（text-sm），颜色 text-slate-500，分隔符用 `/`
- 页面标题：text-2xl font-semibold text-slate-900
- 主操作按钮：页面标题同行，右对齐

**适用页面：** 凭证列表、科目表、客户档案、销售发票、采购发票、资产清单、合并组列表、汇率管理等

**设计要求：**

- 金额列：右对齐，使用等宽字体（tabular-nums）
- 复选框：支持批量选择 + 批量操作（如批量过账）
- 排序：点击列标题切换升/降序，排序列标题加箭头图标
