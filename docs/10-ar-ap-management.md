# 应收账款与应付账款管理

本文档描述系统中应收账款（AR）和应付账款（AP）子账本的业务规则，遵循**中国企业会计准则 CAS 22《金融工具确认和计量》**及相关准则，帮助财务团队管理企业的往来账目、发票核销和坏账处理。

---

## 1. 业务概述

### 应收账款（AR）

应收账款是企业因向客户销售商品或提供服务，尚未收到货款而产生的债权。系统需要在总账的基础上，维护每个客户的**子账本**（Subledger），记录每笔交易的发票、付款和未结余额。

### 应付账款（AP）

应付账款是企业因向供应商采购商品或接受服务，尚未付款而产生的债务。系统需要维护每个供应商的子账本，记录每笔采购发票、付款和未结余额。

### AR/AP 子账本与总账的关系

- **子账本**：按每个客户/供应商记录明细，包含每张发票和每笔付款的详细信息
- **总账**：汇总所有客户/供应商的余额，体现在资产负债表的"应收账款"和"应付账款"科目
- **关键约束**：AR/AP 子账本余额合计必须始终等于对应的总账科目余额

## 2. 客户档案管理

### 客户档案（Customer Master）

每个客户在系统中有唯一的档案记录，包含其基本信息和开票信息：

| 信息项 | 说明 |
|--------|------|
| 客户编号 | 系统内唯一标识（如 C-00001） |
| 客户名称 | 法定公司名称（与发票抬头一致） |
| 纳税人识别号 | 18位统一社会信用代码（开具增值税专票时必填） |
| 开票地址 | 增值税发票上的注册地址 |
| 开票电话 | 增值税发票上的联系电话 |
| 开户银行 | 增值税发票上的开户行名称 |
| 银行账号 | 增值税发票上的银行账号 |
| 信用期限 | 约定的付款期限（如：30天、60天） |
| 信用额度 | 允许赊销的最高未结余额 |
| 应收账款科目 | 该客户对应的明细科目（如 1122-00001） |
| 是否有效 | 是否仍在使用 |

### 业务规则

- 同一公司内，客户编号不可重复
- 向该客户开具增值税专用发票时，纳税人识别号为必填项
- 信用额度管理（可选）：录入新销售订单时，系统检查是否超过信用额度，超限时警告

## 3. 供应商档案管理

### 供应商档案（Vendor Master）

| 信息项 | 说明 |
|--------|------|
| 供应商编号 | 系统内唯一标识（如 V-00001） |
| 供应商名称 | 法定公司名称 |
| 纳税人识别号 | 18位统一社会信用代码（核验进项发票时使用） |
| 增值税纳税人类型 | 一般纳税人 或 小规模纳税人（决定能否取得专票） |
| 开票地址 | 供应商的注册地址 |
| 开户银行及账号 | 付款时使用 |
| 付款条件 | 约定的付款期限（如：见票即付、月结30天） |
| 应付账款科目 | 该供应商对应的明细科目（如 2202-00001） |
| 是否有效 | 是否仍在使用 |

## 4. 发票管理

### 4.1 销售发票（应收账款侧）

销售发票是企业向客户开具的增值税发票，是确认收入和形成应收账款的依据：

**销售发票包含的信息：**

| 信息项 | 说明 |
|--------|------|
| 发票编号 | 系统内部编号（如 INV-2025-00001） |
| 发票类型 | 增值税专用发票 / 增值税普通发票 |
| 发票代码/号码 | 税务机关分配的发票代码和号码 |
| 开票日期 | 发票开具日期 |
| 客户 | 购买方名称和税号 |
| 明细行 | 商品/服务名称、数量、单价、税率、税额、合计金额 |
| 不含税金额 | 确认为收入的金额 |
| 增值税额 | 销项税额（计入应交税费） |
| 含税合计 | 客户应付的全部金额 |
| 到期日 | 按信用期限计算的最晚收款日期 |
| 关联凭证 | 与该发票对应的日记账凭证 |
| 核销状态 | 未核销、部分核销、已核销 |

### 4.2 采购发票（应付账款侧）

采购发票是企业从供应商收到的增值税发票，是形成应付账款和确认进项税额的依据：

**采购发票包含的信息：**

| 信息项 | 说明 |
|--------|------|
| 发票编号 | 系统内部录入编号（如 PIN-2025-00001） |
| 发票类型 | 增值税专用发票 / 增值税普通发票 |
| 供应商发票号码 | 供应商发票上的代码和号码 |
| 收票日期 | 收到发票的日期 |
| 供应商 | 销售方名称和税号 |
| 不含税金额 | 确认为成本/费用的金额 |
| 增值税额 | 可抵扣进项税额（含税发票且为一般纳税人时） |
| 认证状态 | 待认证 / 已认证 / 认证失败 |
| 到期日 | 按付款条件计算的最晚付款日期 |
| 核销状态 | 未核销、部分核销、已核销 |

## 5. 收付款与核销

### 5.1 核销（Matching）的含义

核销是将**付款记录**与**发票记录**相互匹配的过程，用于跟踪每张发票的实际收付状态：

- 一张发票可以通过一次或多次付款完成核销
- 一次付款可以核销一张或多张发票
- 核销后，发票的"未结余额"相应减少

### 5.2 应收账款核销流程

**收到客户款项：**

1. 录入收款凭证（借：银行存款，贷：应收账款）
2. 在应收账款子账本中，将此次收款与对应的销售发票核销
3. 系统更新发票核销状态和客户未结余额

**核销状态：**

| 状态 | 说明 |
|------|------|
| 未核销 | 发票金额全部未收到 |
| 部分核销 | 已收部分款项，仍有未结余额 |
| 已核销 | 发票金额已全部收到 |

### 5.3 应付账款核销流程

**支付供应商款项：**

1. 录入付款凭证（借：应付账款，贷：银行存款）
2. 在应付账款子账本中，将此次付款与对应的采购发票核销
3. 系统更新发票核销状态和供应商未结余额

### 5.4 预收/预付款项处理

**预收款：**
- 收到客户预付款时：借：银行存款，贷：预收款项（2203）
- 开票发货后：借：预收款项，贷：主营业务收入+应交税费

**预付款：**
- 向供应商预付款时：借：预付款项（1123），贷：银行存款
- 收到发票和货物后：借：相关资产/费用+应交税费，贷：预付款项

## 6. 账龄分析

账龄分析是将未结应收/应付账款按拖欠时间长短分组，评估回收风险或付款及时性。

### 标准账龄分组

| 账龄区间 | 说明 |
|---------|------|
| 未到期 | 尚未超过信用期限的正常账款 |
| 逾期 1–30 天 | 超过付款期限不足30天 |
| 逾期 31–60 天 | 超过付款期限31至60天 |
| 逾期 61–90 天 | 超过付款期限61至90天 |
| 逾期 91–180 天 | 超过付款期限91至180天 |
| 逾期 181–365 天 | 超过付款期限超过半年 |
| 逾期 1 年以上 | 逾期超过一年，回收风险极高 |

### 账龄分析报表

账龄分析报表在期末生成，按客户或供应商分行展示：
- 各账龄区间未结金额
- 合计未结余额
- 占总应收（应付）的比例

## 7. 坏账准备（CAS 22 预期信用损失模型）

### 坏账准备的含义

坏账准备是为应收账款中预期无法收回的部分预先计提的损失准备，反映应收账款的实际可回收价值。

### 预期信用损失（ECL）模型

依据 CAS 22《金融工具确认和计量》，应收账款采用**预期信用损失模型**计提坏账准备：

**简化方法（适用于应收账款）：**
- 在整个存续期内，按照预期信用损失的金额，计提坏账准备
- 无需判断信用风险是否显著增加

### 坏账准备计提方法

**方法一：账龄分析法（最常用）**

根据账龄分组，设定不同的预期损失率：

| 账龄区间 | 参考损失率 | 说明 |
|---------|-----------|------|
| 未到期 | 1%–3% | 整体信用风险低 |
| 逾期 1–30 天 | 5% | 轻微逾期 |
| 逾期 31–90 天 | 10%–20% | 存在一定回收风险 |
| 逾期 91–180 天 | 30%–50% | 回收存在较大不确定性 |
| 逾期 181–365 天 | 50%–80% | 回收困难 |
| 逾期 1 年以上 | 80%–100% | 基本无法收回 |

**注：** 具体损失率应根据企业历史坏账数据和所在行业特点确定，上表为参考范围，企业须制定自己的坏账计提政策。

**方法二：个别认定法（针对重大单项应收款）**

对于金额重大的单项应收款，逐笔评估其可回收性，单独计提坏账准备。

### 坏账准备会计分录

**计提坏账准备（期末）：**
- 借：信用减值损失（资产减值损失）
- 贷：坏账准备（1122的备抵科目）

**实际发生坏账（核销无法收回的账款）：**
- 借：坏账准备
- 贷：应收账款——某客户

**已核销款项后续收回：**
- 先恢复应收账款：
  - 借：应收账款——某客户
  - 贷：坏账准备
- 再确认收款：
  - 借：银行存款
  - 贷：应收账款——某客户

## 8. 月末应收/应付核查流程

在每个会计期间关闭前，应完成以下核查：

### 应收账款核查清单

| 核查项目 | 说明 |
|---------|------|
| 子账本与总账核对 | 所有客户应收余额合计 = 应收账款总账科目余额 |
| 账龄分析 | 生成当期账龄报表，关注大额逾期账款 |
| 坏账准备测算 | 按账龄分析法计算应计提金额，与现有准备余额对比 |
| 差异调整 | 若坏账准备不足，补提；若计提过多，冲回 |
| 客户余额函证（年末） | 年末向重要客户发送余额确认函，核实应收金额 |

### 应付账款核查清单

| 核查项目 | 说明 |
|---------|------|
| 子账本与总账核对 | 所有供应商应付余额合计 = 应付账款总账科目余额 |
| 到期付款提醒 | 检查即将到期的应付账款，安排付款资金 |
| 未到票发票确认 | 确认货物已收到但发票尚未到达的采购，是否需要暂估入账 |
| 供应商对账 | 定期与重要供应商核对往来账目，消除差异 |

## 9. 暂估入账

**适用场景：** 月末，货物已经收到并入库，但供应商发票尚未到达。

**处理方法：**

月末暂估：
- 借：相关资产/费用（暂估金额）
- 贷：应付账款——暂估（专设科目）

下月收到发票时冲回暂估，录入正式发票凭证：
- 借：应付账款——暂估（冲回）
- 贷：相关资产/费用
- 借：相关资产/费用（发票金额）
- 借：应交税费——应交增值税（进项税额）
- 贷：应付账款——供应商

---

## 10. 内部往来账款

集团公司间的往来账款（属于内部往来，而非外部应收应付）应使用专设的**内部往来科目**，而非普通的应收账款或应付账款科目：

- 内部往来科目（如 1601-内部往来——关联公司A）
- 合并报表时，所有内部往来余额须完全抵消
