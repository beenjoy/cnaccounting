# 业务工作流

本文档描述系统中各主要业务场景的完整操作流程，遵循**中国企业会计准则（CAS）**，帮助财务团队了解每项工作从开始到完成的步骤和注意事项。

---

## 1. 日记账凭证录入与审批流程

### 流程概览

日常会计记录通过凭证录入完成，每张凭证需经过创建、审批、过账三个主要阶段。

### 详细步骤

**第一阶段：创建凭证（会计执行）**

1. 选择需要记账的公司
2. 填写凭证基本信息：
   - 交易日期（经济业务实际发生的日期）
   - 说明（描述业务内容的文字，必填）
   - 参考号（外部单据编号，如发票号，可选）
   - 凭证类型（普通、调整、内部往来等）
3. 选择所属会计期间（系统根据交易日期自动推荐）
4. 逐行录入凭证明细：
   - 选择科目
   - 填写借方或贷方金额（二选一）
   - 若涉及外币，填写货币代码和汇率
5. 确认借贷合计相等后，保存为草稿

**第二阶段：提交审批（会计执行）**

6. 检查凭证内容无误后，提交凭证进入审批队列
7. 凭证状态变为"待审批"，不可再编辑

**第三阶段：审批凭证（财务经理或授权人员执行）**

8. 审批人查看待审批凭证列表
9. 审阅凭证内容，确认业务依据和金额
10. 选择以下操作之一：
    - **批准**：凭证进入"已审批"状态，等待过账
    - **拒绝退回**：填写拒绝原因，凭证退回草稿，会计可修改后重新提交

**第四阶段：过账（授权人员执行）**

11. 对已审批的凭证执行过账操作
12. 系统进行五项验证（借贷平衡、科目有效、期间开放、货币一致、内部往来匹配）
13. 验证通过后，系统分配凭证编号（如 JE-2025-00001），凭证正式入账
14. 凭证状态变为"已过账"，相关科目余额随即更新

### 错误更正流程

若已过账凭证存在错误，处理步骤如下：

1. 创建冲销凭证（将原凭证借贷方向互换）
2. 冲销凭证经过完整的审批和过账流程
3. 另行创建正确金额的新凭证，再次走审批和过账流程
4. 原凭证标记为"已冲销"

## 2. 会计期间管理流程

### 月末关闭流程

**执行人：期间管理员 / 财务经理**

1. 确认本月所有业务凭证均已过账（查看待审批队列是否清空）
2. 按照月末核查清单逐项确认（共11项，详见业务规则文档）
3. 完成以下月末调整工作（如适用）：
   - 录入并过账折旧分录
   - 录入并过账预付费用摊销分录
   - 录入并过账应计收入和应计费用分录
   - 如有外币余额，执行外币重估
4. 再次检查试算平衡，确认借贷相等
5. 财务主管审核确认
6. 执行关闭期间操作，期间状态变为"关闭"

### 期间重新开放流程

若在期间关闭后发现需要补录或修正凭证：

1. 填写重开原因（详细说明为何需要重开，不少于10个字符）
2. 执行重开操作，期间恢复为"开放"状态
3. 补录或修正相关凭证并完成过账
4. 重新执行月末关闭流程

**注意**：每次重开操作均自动记录在审计日志中，包含操作人、时间和重开原因。

## 3. 年末结账流程

**执行人：会计总监 / 财务经理**

### 前提条件确认

在执行年末结账前，确认以下条件全部满足：
- 公司已配置未分配利润科目
- 本会计年度全部 **12 个期间**（第1–12期）均已关闭
- 年末审计调整分录已全部过入第12期（12月）

### 结账步骤

**第一步：启动年末结账（Begin Close）**

1. 进入年末结账操作页面，选择需要结账的会计年度
2. 系统显示结账影响预览：
   - 将被归零的收入科目及余额
   - 将被归零的费用科目及余额
   - 转入未分配利润的净额（净利润或净亏损）
3. 确认后，系统自动生成结账凭证（过入第12期）：
   - 将所有收入类科目余额转入未分配利润（收入科目归零）
   - 将所有费用类科目余额从未分配利润中扣减（费用科目归零）
4. 会计年度状态变为"结账中"

**第二步：审查结账凭证**

5. 审阅系统自动生成的结账凭证，确认数字无误
6. 确认未分配利润科目的变动金额等于当年净利润（或净亏损）

**第三步：完成年末结账（Complete Close）**

7. 执行完成结账操作
8. 系统将结账凭证自动过账
9. 会计年度状态变为"已关闭"
10. 此后，该年度不可再录入任何凭证

**第四步：提取法定盈余公积（依据中国公司法）**

11. 年末结账完成后，财务经理发起盈余公积计提：
    - 计提金额 = 当年净利润 × 10%
    - 若法定盈余公积累计金额已达注册资本 50%，可不再计提
12. 录入计提凭证：借记"未分配利润"，贷记"盈余公积（法定盈余公积）"
13. 凭证经审批过账后，未分配利润减少，盈余公积相应增加

**可选第五步：生成新年度期初分录**

14. 创建新会计年度（下一年度）
15. 将当年资产负债表科目余额复制为新年度期初余额（生成期初凭证）

## 4. 合并报表编制流程

**执行人：合并专员 / 财务经理**

### 合并前准备

1. 确认所有参与合并的成员公司均已完成本期记账和月末关闭
2. 确认所有涉及货币对的折算汇率（期末汇率和平均汇率）均已录入
3. 确认合并组配置正确（成员公司列表、持股比例、合并方法、抵消规则）
4. 对需要完全合并的子公司，检查内部往来账目是否已完成核对

### 合并执行步骤

启动合并运行后，系统按以下 8 个步骤自动处理（每步完成后才进入下一步）：

| 步骤 | 系统操作 | 人工关注点 |
|------|---------|----------|
| 1. 验证成员数据 | 检查所有成员公司目标期间已关闭，试算表平衡 | 如有验证错误，先修复后重新运行 |
| 2. 货币折算 | 按 CAS 19《外币折算》将各公司数据折算为报告货币 | 确认汇率选用正确 |
| 3. 汇总余额 | 将各成员折算后的科目余额加总 | — |
| 4. 内部往来匹配 | 自动比对集团内部公司间往来差异 | 查看未匹配或差异项，判断是否需要处理 |
| 5. 生成抵消分录 | 按配置的抵消规则自动生成消除凭证，过账到合并目标期间 | 检查抵消分录是否完整、金额是否正确 |
| 6. 计算非控制性权益 | 按各子公司非控制性权益比例计算应归属的净资产和净利润 | 确认非控制性权益比例配置正确 |
| 7. 生成合并试算表 | 汇总得出合并后的完整试算平衡表 | 验证合并试算表借贷相等 |
| 8. 最终验证 | 对合并结果进行完整性检查，生成验证报告 | 查看警告信息，评估是否影响报表质量 |

### 合并完成后

- 查看并打印五大合并财务报表（资产负债表、利润表、现金流量表、所有者权益变动表）
- 审阅非控制性权益披露
- 审阅内部往来差异说明（若有未能完全抵消的差异）

## 5. 外币重估流程

**执行人：会计 / 财务经理**

### 适用时机

每个会计期间结束时，若公司持有以外币计价的货币性资产或负债（外币现金、外币应收账款、外币贷款等），需执行外币重估。

### 重估步骤

1. 确认期末汇率已录入系统（折算日期 = 期末日期，类型 = 期末汇率）
2. 进入外币重估页面，选择目标公司和期间
3. 选择重估方法：
   - **全资产负债法**：对所有货币性资产和负债进行重估
   - **仅未结项目法**：仅对尚未结清的应收/应付项目重估
4. 系统自动计算各外币科目的：
   - 外币余额
   - 重估前记账本位币金额
   - 重估后记账本位币金额（= 外币余额 × 期末汇率）
   - 汇兑差额金额
5. 审阅重估结果，确认汇率和科目正确
6. 确认执行，系统自动生成重估凭证并过账到当期
7. 依据 CAS 19，汇兑差额计入当期**财务费用**（差额为正则减少财务费用，为负则增加财务费用）

**注意**：每个期间只能执行一次重估。若需重跑，须先撤销原重估记录。

## 6. 成员邀请与加入流程

### 邀请新成员（管理员执行）

1. 进入组织成员管理页面
2. 点击邀请成员，填写受邀人的邮箱地址
3. 选择为受邀人分配的基础角色（所有者/管理员/成员/查看者）
4. 可同时勾选职能角色（如会计、期间管理员等）
5. 发送邀请，系统向该邮箱发送邀请邮件（邀请链接有效期通常为7天）

### 受邀人接受邀请

6. 受邀人收到邮件，点击邀请链接
7. 若尚未注册系统账号，先完成注册
8. 已有账号则直接登录并确认接受邀请
9. 接受后，受邀人自动成为该组织的成员，获得对应角色权限

### 邀请管理

- 未过期的待接受邀请可由管理员撤销
- 同一邮箱对同一组织只能有一条待接受的邀请，不可重复发送
- 邀请过期后自动失效，可重新发送新邀请

## 7. 科目表维护流程

### 新建科目

**执行人：财务经理**

1. 确定新科目的业务用途
2. 选择科目所属的公司
3. 填写科目信息：
   - 科目编号（公司内唯一，遵循财政部统一科目编码规范）
   - 科目名称
   - 科目类型（资产/负债/所有者权益/收入/费用）
   - 科目类别（如流动资产、主营业务收入等）
   - 是否为可过账科目（末级科目通常可过账，汇总科目通常不可过账）
   - 若适用，指定父科目（父科目类型必须与新科目类型一致）
4. 保存，新科目即可用于凭证录入

### 停用科目

**前提条件**：科目余额为零，且不被任何抵消规则引用

1. 确认科目当前余额为零
2. 执行停用操作，科目标记为不活跃
3. 停用后不可在新凭证中使用，但历史记录保留可查

### 集团科目表映射

1. 在组织层级配置集团科目表（标准科目体系）
2. 为每家公司的本地科目建立与集团科目的映射关系
3. 映射关系支持三种方式：
   - 直接映射（一个本地科目对应一个集团科目）
   - 范围映射（一段编号范围内的本地科目对应同一集团科目）
   - 规则映射（按编号规则匹配到集团科目）
4. 配置完成后，合并时系统可依据映射关系汇总跨公司的科目数据

## 8. 汇率录入流程

**执行人：财务经理 / 会计**

### 手动录入汇率

1. 进入汇率管理页面
2. 选择来源货币和目标货币
3. 选择汇率类型（期末汇率 / 平均汇率 / 历史汇率）
4. 填写生效日期和汇率值
5. 保存，汇率即可在凭证录入和报表折算中使用

### 从外部数据源同步汇率

系统支持从欧洲央行（通过 Frankfurter 数据服务）自动同步主要货币对的每日汇率：

1. 点击"从外部同步汇率"按钮
2. 系统自动获取最新汇率数据并导入
3. 对于系统无法直接获取的货币对，系统通过三角计算自动推算（以美元或欧元为中间货币）
4. 查看同步结果，确认数据正确

**注意**：
- 同步导入的汇率标记为"系统汇率（ECB）"，与手动录入的汇率区分显示
- 同一货币对、同一日期、同一类型的汇率不可重复录入

## 9. 销售发票开具与应收账款核销流程

**执行人：财务会计（AR 岗）**

### 9.1 开具销售发票并确认收入

1. 销售业务达成后，进入"销售发票"模块，新建销售发票
2. 选择客户档案（系统自动带入客户税号、开票信息）
3. 填写发票明细行：商品/服务名称、数量、单价、适用税率
4. 系统自动计算：不含税金额、税额、含税合计
5. 确认开票日期，系统按信用期限自动推算到期日
6. 提交发票，系统自动生成过账凭证：
   - 借：应收账款——某客户（含税合计）
   - 贷：主营业务收入（不含税金额）
   - 贷：应交税费——应交增值税（销项税额）
7. 同时生成 VATRecord（方向：销项），关联至本张发票

### 9.2 收到客户款项并执行核销

1. 收到客户付款后，录入收款凭证：
   - 借：银行存款
   - 贷：应收账款——某客户
2. 在 AR 子账本中打开收款核销界面，选择本次付款对应的发票
3. 填写每张发票的核销金额（允许一次性核销多张发票）
4. 确认核销，系统生成 ARMatching 记录，并更新发票核销状态
5. 系统实时检查：子账本余额合计 = 总账应收账款科目余额

### 9.3 月末应收账款核查

1. 运行账龄分析报表，按7个账龄区间查看未结应收余额
2. 关注大额逾期账款，与客户沟通确认收款安排
3. 按账龄分析法计算本期应计提坏账准备金额，与现有准备余额对比
4. 若需补提：录入坏账准备补提凭证（借：信用减值损失，贷：坏账准备）
5. 确认子账本合计余额与总账科目余额一致

## 10. 采购发票录入与应付账款核销流程

**执行人：财务会计（AP 岗）**

### 10.1 录入采购发票并确认应付

1. 收到供应商发票后，进入"采购发票"模块，新建采购发票
2. 选择供应商档案（系统自动带入供应商税号、纳税人类型）
3. 填写发票信息：发票类型、供应商发票号码、收票日期、税率、金额
4. 系统按付款条件自动推算到期日
5. 提交录入，系统自动生成过账凭证：
   - 借：原材料/库存商品/相关费用（不含税金额）
   - 借：应交税费——应交增值税（进项税额）[仅一般纳税人且为专票时]
   - 贷：应付账款——某供应商（含税合计）
6. 同时生成 VATRecord（方向：进项，认证状态：待认证）

### 10.2 增值税专票认证

1. 对于增值税专用发票，在认证期限内（通常为收票次月底前）进行认证
2. 与税务系统核验发票真实性和合规性
3. 认证通过后，将 VATRecord 认证状态更新为"已认证"，该进项税额可参与当月抵扣
4. 认证失败的发票须核查原因，进项税额暂时不可抵扣

### 10.3 支付供应商款项并执行核销

1. 在发票到期前安排付款，录入付款凭证：
   - 借：应付账款——某供应商
   - 贷：银行存款
2. 在 AP 子账本中执行核销，将本次付款与对应的采购发票关联
3. 系统生成 APMatching 记录，更新发票核销状态
4. 系统实时检查：子账本余额合计 = 总账应付账款科目余额

### 10.4 月末应付账款核查

1. 查看即将到期（7天内）的未付发票，安排付款资金
2. 对货物已收到但发票未到的情形，执行暂估入账：
   - 月末暂估：借：相关资产/费用（暂估金额），贷：应付账款——暂估
   - 次月收到发票后先冲回：借：应付账款——暂估，贷：相关资产/费用
   - 再录入正式发票凭证（含进项税）
3. 与重要供应商定期对账，消除账目差异
4. 确认子账本合计余额与总账科目余额一致

## 11. 月度增值税结算与申报流程

**执行人：税务会计 / 财务经理**

**时间安排：** 每月（或每季度）申报截止日前（通常为次月15日）

### 第一步：汇总当期增值税数据

1. 进入"增值税管理"模块，选择申报期间
2. 系统汇总本期所有已过账的 VATRecord：
   - 销项税额合计（所有销售发票的税额之和）
   - 进项税额合计（所有已认证、可抵扣的进项 VATRecord 税额之和）
3. 生成 VATDeclaration 草稿，核查数据是否完整

### 第二步：计算应纳税额

```
应纳税额 = 本期销项税额合计 - 本期可抵扣进项税额合计
```

- **应纳税额 > 0**：需要缴税，进入第三步
- **应纳税额 ≤ 0**：产生留抵税额，结转至下期，本月无需缴税，直接申报

### 第三步：过账增值税结转凭证（应纳税额 > 0 时）

录入"转出未交增值税"凭证：
- 借：应交税费——应交增值税（转出未交增值税）
- 贷：应交税费——未交增值税

### 第四步：计算并过账附加税凭证

以本期实际应缴纳增值税额为基数：
- 借：税金及附加（5201）
- 贷：应交税费——城建税
- 贷：应交税费——教育费附加
- 贷：应交税费——地方教育附加

### 第五步：申报并缴税

1. 生成增值税申报汇总表（辅助核对税务申报表数据）
2. 在税务机关系统（网上申报平台）完成增值税申报
3. 银行缴税后，录入缴税凭证：
   - 借：应交税费——未交增值税
   - 贷：银行存款
4. 同步录入附加税缴税凭证：
   - 借：应交税费——城建税（教育费附加、地方教育附加）
   - 贷：银行存款
5. 将 VATDeclaration 状态更新为"已缴税"

## 12. 固定资产取得与建账流程

**执行人：资产管理岗 / 财务会计**

### 12.1 外购固定资产

1. 收到资产并取得发票后，录入采购凭证：
   - 借：固定资产（1601-资产类别-资产名称）
   - 借：应交税费——应交增值税（进项税额）[一般纳税人且符合抵扣条件时]
   - 贷：应付账款——供应商 或 银行存款
2. 进入"固定资产"模块，新建资产卡片（FixedAsset），填写：
   - 资产编号（系统自动生成）、名称、类别
   - 取得日期、原值（与凭证金额一致）
   - 预计净残值率、折旧方法、预计使用年限
   - 使用部门（决定折旧费用科目）
3. 保存后，系统自动：
   - 计算月折旧额，生成折旧计划（DepreciationSchedule）
   - 确认折旧起始月份（取得次月）

### 12.2 在建工程转入固定资产

1. 在建工程完工并可正式使用时，录入转账凭证：
   - 借：固定资产（竣工决算价值）
   - 贷：在建工程（1604）
2. 在系统中新建资产卡片，取得日期填写正式交付使用日期

## 13. 月度折旧计提流程

**执行人：资产管理岗 / 财务会计**

**时间安排：** 每月末（会计期间关闭前）

### 第一步：确认本期应计提折旧的资产范围

系统筛选满足以下条件的资产：
- 资产状态为"正常使用"或"停用"
- 本月为折旧起始月份或之后的月份
- 尚未达到预计净残值（累计折旧 < 原值 - 净残值）

### 第二步：批量计算本期折旧额

1. 触发"批量计提折旧"操作，选择公司和会计期间
2. 系统按各资产的折旧方法，分别计算本期折旧额：
   - 直线法：月折旧额 = （原值 - 净残值）÷ 总月数
   - 加速折旧法：按折旧计划中本期对应金额
   - 工作量法：单位折旧额 × 本月实际工作量（需手动录入工作量数据）
3. 系统汇总各部门的折旧总额，生成待确认的折旧凭证列表

### 第三步：确认并过账折旧凭证

1. 财务人员核查折旧凭证列表，确认无误
2. 批量过账所有折旧凭证，示例分录：
   - 借：制造费用——折旧费（生产设备）
   - 借：管理费用——折旧费（办公设备）
   - 借：销售费用——折旧费（销售用设备）
   - 贷：累计折旧（1602）
3. 系统更新各资产的累计折旧余额和当前账面净值
4. 生成 DepreciationRecord，关联至本期凭证

### 第四步：期后核查

1. 运行固定资产折旧汇总表，核查本期折旧总额与预计是否一致
2. 检查是否有新增资产未纳入折旧计提范围
3. 确认已处置资产本期不再计提折旧
