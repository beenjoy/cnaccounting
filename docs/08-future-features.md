## P1 — 凭证期间选择优化

**背景：** 目前凭证录入时，用户填写交易日期后，系统不会主动引导用户确认对应的会计期间，用户可能误选期间（尤其是跨月业务或临近月末的交易）。

**目标体验：**

在凭证日期输入框旁增加期间选择增强功能：
- 输入交易日期后，自动显示对应的会计期间名称（如"2025年12月 - 第12期"）
- 期间旁显示状态标识（开放 / 已关闭）
- 若对应期间已关闭，显示警告提示，提示用户该期间需重开后才能过账
- 若选择的期间已关闭，在提交前给出明确警告（不阻止填写，但阻止提交）

**预估复杂度：** 中等
**预估影响：** 显著降低期间选错的概率，减少因过错期间导致的凭证退回

## P2 — 汇率自动同步

**背景：** 目前所有汇率均需财务人员手动录入，工作量大且容易遗漏。

**目标功能：**

从欧洲央行（通过 Frankfurter 免费数据服务）自动获取每日主要货币对的官方汇率：

- 每日自动同步一次（欧洲央行汇率更新时间后）
- 同步失败时发送告警通知
- 支持手动触发同步（不必等待每日自动同步）
- 对于数据源未直接提供的货币对，通过三角计算自动推算（以美元或欧元为中间货币）

**三角计算示例：**
- 已知：美元/日元 = 155.0，美元/人民币 = 7.2
- 推算：日元/人民币 = 7.2 ÷ 155.0 = 0.0465

**界面展示：**
- 汇率列表区分显示"系统汇率（ECB来源）"和"手动录入"
- 显示最后同步时间和同步状态
- 提供同步历史记录查看

**预估复杂度：** 高
**预估影响：** 大幅减少汇率手工维护工作量，提高汇率数据及时性

## P2 — 权限策略管理体验改进

**背景：** 当前权限策略配置界面存在以下不足：
1. 每条策略只能针对一种资源类型，无法同时对多种资源（如同时限制凭证和科目）生效
2. 没有用户级别的策略配置入口（只能配置角色级策略）
3. 配置流程不够直观

**目标设计：**

**两个配置入口：**

| 入口 | 位置 | 用途 |
|------|------|------|
| 策略管理页 | 组织设置 → 权限策略 | 配置适用于角色的通用规则 |
| 成员详情页 | 组织设置 → 成员 → 某位成员 | 配置针对该用户的个性化例外 |

**四步引导式配置界面：**

第一步 — **谁（主体）**：选择策略适用的角色或特定用户

第二步 — **做什么（操作）**：按业务模块分组选择操作权限
  - 日记账凭证：创建、提交、审批、过账
  - 科目管理：创建、修改、停用
  - 快速选择：只读 / 标准 / 完全访问

第三步 — **对什么（资源范围）**：可多选资源类型，并按公司过滤
  - 勾选：日记账凭证
  - 勾选：会计科目
  - 过滤条件：限定特定公司

第四步 — **效果**：允许 或 拒绝
  - 可选高级条件：时间限制（工作日/工作时间）

**数据模型变更（业务说明）：**
- 当前：每条策略只能针对单一资源类型
- 目标：每条策略可同时覆盖多种资源类型，减少需要配置的策略条数

**预估复杂度：** 高
**预估影响：** 权限配置更灵活，减少配置工作量，降低误配置风险

## P3 — 集团科目表（Group Chart of Accounts）

**背景：** 目前每家公司维护各自独立的科目表，在进行集团合并报表时，不同公司可能使用不同科目编号来记录性质相同的业务（如 A 公司用1010记录现金，B 公司用1001记录现金），系统无法自动汇总，需要手工对应，工作量大且容易出错。

**目标架构：两层科目体系**

**第一层：集团科目表（组织级）**

由集团财务统一定义的标准科目体系，是所有公司的共同参照基准。每个集团科目有固定的编号、名称和类型。

示例：
- 集团科目 1000：现金及等价物
- 集团科目 1100：应收账款

**第二层：公司科目表（公司级）**

各公司维护自己的本地科目，并与集团科目建立映射关系。

示例：
- A 公司 1010（活期存款）→ 对应集团科目 1000（现金及等价物）
- A 公司 1020（定期存款）→ 对应集团科目 1000（现金及等价物）
- A 公司 1110（贸易应收款）→ 对应集团科目 1100（应收账款）

**科目映射方式（三种）：**

| 映射方式 | 适用场景 |
|---------|---------|
| 直接映射 | 一个本地科目精确对应一个集团科目 |
| 范围映射 | 某一编号段内的所有本地科目对应同一个集团科目 |
| 规则映射 | 按编号规则（正则匹配）映射到集团科目 |

**主要收益：**
- 合并时系统可自动按集团科目汇总跨公司数据
- 各公司可保留自己的本地科目体系，无需统一编号
- 在集团层面生成标准化的合并报表

**预估复杂度：** 非常高（涉及科目体系架构的根本性变更）
**预估影响：** 大幅简化合并报表编制工作，提高跨公司数据可比性
