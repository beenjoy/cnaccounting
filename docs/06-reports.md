# 财务报告

本文档描述系统生成的各类财务报表的业务用途、呈现内容和使用说明，遵循**中国企业会计准则（CAS）**，帮助财务团队了解每张报表的含义和决策价值。

---

## 报表查询方式

所有财务报表均按**会计期间**查询，而非按任意日期范围。查询时需要指定会计年度和期间编号（1–12），系统将计算该期间截止日的财务数据。

可选择以下查询条件：
- **报告货币**：报表展示时使用的货币（默认为公司记账本位币）
- **对比期间**：可同时显示当期与前1–4个期间的数据进行横向对比
- **是否包含零余额科目**：默认不显示余额为零的科目

报表支持以下格式输出：屏幕查看（在线格式）、PDF（适合打印和存档）、Excel（适合进一步分析）、CSV（适合数据导出）。

## 1. 试算平衡表（Trial Balance）

### 业务用途

试算平衡表是财务人员内部使用的核查工具，用于验证账簿中所有科目的借方合计是否等于贷方合计。它是生成其他财务报表前的必要步骤，也是期末关账前的标准核查项目。

**使用场景：**
- 月末/年末关账前验证账簿准确性
- 发现记账错误或遗漏
- 为编制其他报表提供数据基础

### 报表内容

试算平衡表列示所有有余额的会计科目，每个科目一行，显示：
- 科目编号和名称
- 科目类型和类别
- 期末借方余额
- 期末贷方余额

表格最后显示全部科目的借方合计和贷方合计，两者**必须相等**。若不相等，表明账簿存在错误，系统将发出警告，并建议在试算不平衡的情况下暂缓生成其他报表。

## 2. 资产负债表（Balance Sheet）

**遵循准则：** CAS 30《财务报表列报》

### 业务用途

资产负债表反映企业在某一**特定时间点**（期末日）的财务状况，是企业"此刻拥有什么、欠了多少、净值多少"的快照。

**使用场景：**
- 评估企业的短期偿债能力（流动比率）
- 评估企业的长期财务稳健性
- 向银行、投资方、监管机构提供财务状况证明
- 股东了解企业净资产

### 报表结构

资产负债表分为三大区块，遵循恒等式：**资产 = 负债 + 所有者权益**

**资产部分：**

| 分类 | 包含内容 |
|------|---------|
| 流动资产 | 货币资金、交易性金融资产、应收票据、应收账款、预付款项、其他应收款、存货、其他流动资产（一年内可变现） |
| 非流动资产 | 固定资产、在建工程、无形资产、长期股权投资、长期待摊费用、递延所得税资产 |

**负债部分：**

| 分类 | 包含内容 |
|------|---------|
| 流动负债 | 短期借款、应付票据、应付账款、预收款项、应付职工薪酬、应交税费、其他应付款（一年内须偿还） |
| 非流动负债 | 长期借款、应付债券、长期应付款、递延所得税负债（一年以上到期） |

**所有者权益部分：**

| 项目 | 说明 |
|------|------|
| 实收资本（或股本） | 股东历次投入的注册资本 |
| 资本公积 | 超出注册资本的出资额及其他资本性收益 |
| 盈余公积 | 依法从净利润中提取的法定积累（法定盈余公积） |
| 未分配利润 | 历年累计净利润中未分配的部分（扣除盈余公积提取和股利分配后） |
| 其他综合收益 | 境外子公司报表折算差额等不经过利润表的收益/损失 |
| 非控制性权益 | 仅合并报表：子公司中不属于母公司的权益部分 |

### 验证规则

报表最底端验证：**资产总计 = 负债总计 + 所有者权益总计**。若此等式不成立，表明数据存在问题。

### 对比期间

可同时展示本期末与上期末的数据，帮助使用者了解财务状况的变化趋势。

## 3. 利润表（Income Statement）

**遵循准则：** CAS 30《财务报表列报》

### 业务用途

利润表反映企业在某一**会计期间**（通常为年度或月度）的经营成果，展示企业在该期间赚取了多少收入、发生了多少成本和费用，最终实现了盈利还是亏损。

**使用场景：**
- 评估企业盈利能力和经营效率
- 对比不同期间的经营趋势
- 计算各类利润率指标（毛利率、营业利润率、净利润率）
- 向管理层和股东汇报经营成果

### 报表格式（CAS 30 标准格式）

利润表按以下层次分步展示各层利润：

**第一层：营业收入**
- 主营业务收入
- 其他业务收入

**第二层：营业成本**
- 主营业务成本
- 其他业务成本

**第三层：期间费用与税费**
- 税金及附加（增值税附加、消费税、城市维护建设税等）
- 销售费用
- 管理费用
- 研发费用
- 财务费用（含利息支出/收入、汇兑差额）

**第四层：其他收益项目**
- 其他收益（政策性补贴等）
- 投资收益（或损失）
- 资产减值损失（以负数列示）
- 资产处置收益（或损失）

**第五层：【营业利润】**（中国特有指标）

= 营业收入 - 营业成本 - 税金及附加 - 期间费用 + 其他收益 ± 投资收益 - 资产减值损失 ± 资产处置收益

**第六层：营业外项目**
- 营业外收入
- 营业外支出

**第七层：【利润总额】**（税前利润）

= 营业利润 + 营业外收入 - 营业外支出

**第八层：所得税费用**

**最终：净利润（或净亏损）**

= 利润总额 - 所得税费用

对于合并报表，净利润下方还需分别列示：
- 归属于母公司所有者的净利润
- 归属于非控制性权益的净利润

**综合收益总额**

= 净利润 + 其他综合收益（税后净额）

对于合并报表，综合收益总额下方分别列示：
- 归属于母公司所有者的综合收益总额
- 归属于非控制性权益的综合收益总额

### 时间范围选择

利润表可按以下时间范围生成：
- **年初至今（默认）**：从该会计年度第1期至当前期间的累计数
- **当期数据**：仅当前期间的收入和费用

### 每股收益（仅合并报表）

若合并组已设置总股本数量，利润表底部可附加每股收益（EPS）披露：
- 基本每股收益 = 归属于母公司所有者的净利润 ÷ 加权平均普通股股数
- 稀释每股收益 = 归属于母公司所有者的净利润 ÷ 稀释后加权平均股数

## 4. 现金流量表（Cash Flow Statement）

**遵循准则：** CAS 31《现金流量表》

### 业务用途

现金流量表反映企业在某一会计期间**实际现金的流入和流出**，回答"企业的现金去了哪里、从哪里来"的问题。这与利润表不同——企业可以有利润但现金紧张，现金流量表帮助揭示这种情况。

**使用场景：**
- 评估企业的实际现金生成能力
- 分析资本支出和投资活动
- 了解融资活动（借款、还款、股利）
- 预测短期资金需求

### 三大分类

现金流量表将所有现金流动按经济性质分为三类：

**经营活动现金流**
- 与企业核心经营业务相关的现金流入和流出
- 包括：收到的客户付款、支付的供应商货款、支付的薪资、支付的税款和利息等

**投资活动现金流**
- 与长期资产购置、处置和投资相关的现金流
- 包括：购买固定资产、出售资产所得、长期投资的购置和回收等

**融资活动现金流**
- 与资本结构和分配相关的现金流
- 包括：取得贷款、偿还贷款、股东分红、股票发行所得、回购股票支出等

### 两种编制方法

**间接法（默认，推荐）**

从净利润出发，通过调整非现金项目和营运资本变动，推导出经营活动现金流：
- 加回折旧、摊销等非现金费用
- 加减应收账款、存货、应付账款等营运资本变动

**直接法**

直接列示各主要现金收付类别的实际金额：
- 收自客户的现金
- 付给供应商的现金
- 付给员工的薪资
- 支付的税款和利息

两种方法的经营活动现金净额结果相同。

### 报表结构

- 经营活动产生的现金流量净额
- 投资活动产生的现金流量净额
- 融资活动产生的现金流量净额
- 汇率变动对现金及现金等价物的影响（适用于持有外币的公司）
- **= 现金及现金等价物净增加额**
- 加：期初现金及现金等价物余额
- **= 期末现金及现金等价物余额**（应与资产负债表上的货币资金余额一致）

### 必要披露

根据 CAS 31 要求，现金流量表须附加以下信息：
- 本期实际支付的利息金额
- 本期实际支付的所得税金额
- 重大非现金投融资活动（如以融资租赁方式获取设备，无实际现金流转但资产和负债同时增加）

## 5. 所有者权益变动表（Statement of Changes in Equity）

**遵循准则：** CAS 30《财务报表列报》

> **重要：** 依据 CAS 30，所有者权益变动表是中国企业**强制编制**的财务报表，不可省略。

### 业务用途

所有者权益变动表展示某一会计期间内，所有者权益各项目从期初到期末的全部变化过程。它填补了资产负债表（仅显示期末余额）和利润表（仅显示净利润）之间的信息空缺。

**使用场景：**
- 了解未分配利润的增减变化
- 追踪盈余公积的提取和使用情况
- 追踪股利分配情况
- 了解股票发行和回购活动
- 查看其他综合收益（如境外子公司折算差额）的历史变化

### 报表结构

报表以矩阵形式呈现，行为各类变动事项，列为权益各项目：

**列（权益项目）：**
- 实收资本（或股本）
- 资本公积
- 盈余公积
- 未分配利润
- 其他综合收益
- 非控制性权益（仅合并报表）
- 合计

**行（变动事项）：**

| 变动事项 | 说明 |
|---------|------|
| 期初余额 | 年初各项权益余额 |
| 净利润 | 当期净利润（利润表净利润转入） |
| 其他综合收益 | 境外子公司折算差额、未实现损益等 |
| 提取盈余公积 | 从净利润中依法提取法定盈余公积（净利润的10%，达到注册资本50%后可停止） |
| 宣告股利 | 向股东宣告分配的股利 |
| 股票发行 | 本期新发行股票所得资金 |
| 股票回购 | 本期回购自身股票的支出 |
| 股权激励 | 股票期权或限制性股票费用 |
| 前期差错更正 | 更正以前年度会计错误的影响 |
| 期末余额 | 年末各项权益余额 |

## 6. 合并财务报表

### 与单公司报表的区别

合并报表是将母公司与所有纳入合并范围的子公司财务数据汇总后，消除内部交易影响，以集团整体为单位编制的报表（依据 CAS 33《合并财务报表》）。

主要不同点：
- 需要将各子公司以记账本位币记录的数据折算为集团统一的报告货币（依据 CAS 19《外币折算》）
- 集团内部交易（内部销售、内部借款、内部股利等）需全部抵消
- 资产负债表所有者权益部分单独列示**非控制性权益**
- 利润表净利润下方分别列示归属于母公司所有者和归属于非控制性权益的利润
- 综合收益总额下方同样分别列示两者的份额

### 货币折算差额的处理（CAS 19）

将境外子公司数据折算为报告货币时，因各项目使用不同汇率（期末汇率、平均汇率、历史汇率）而产生的整体折算差额，计入**其他综合收益**，在所有者权益变动表中单独列示。处置境外子公司时，历史积累的折算差额转入当期投资收益（损益）。

### 合并报表的生成前提

合并报表必须基于已成功完成的合并执行生成，不可直接生成而绕过合并执行步骤。

### 查看层次选项

合并报表支持三种查看层次：
- **合并后（默认）**：汇总抵消后的集团整体数据
- **各公司汇总**：各成员公司数据并列展示（未抵消）
- **仅抵消分录**：单独查看本次合并产生的所有抵消分录

## 7. 报表比较与分析

### 对比期间

所有报表均支持显示对比期间数据，帮助使用者快速识别变化趋势：
- 可设置显示前 1–4 个期间的对比数据
- 资产负债表对比上期末余额
- 利润表对比上年同期累计数或上月数

### 差异分析

当显示对比期间时，系统可附加：
- **差异金额**：当期与对比期的绝对差额
- **差异百分比**：变化幅度

## 8. 报表数字显示规范

### 金额显示

- 正数：直接显示（如 150,000.00）
- 负数：PDF格式使用括号（如 (5,000.00)），屏幕格式使用负号（如 -5,000.00）
- 金额按货币规定的小数位显示（人民币：2位，日元：0位）
- 大额数字使用千位分隔符

### 报表行类型

报表中各行根据用途以不同格式呈现：
- **标题行**：粗体，无金额，用于划分区块（如"流动资产"）
- **明细行**：正常字体，显示科目金额
- **小计行**：粗体，行上方有单线，显示分类小计
- **合计行**：粗体，行上方单线、行下方双线，显示大类合计

### 缩进层次

报表行根据在科目层级中的位置显示不同的缩进，直观体现父子关系。

## 9. 应收账款明细与账龄分析表

### 业务用途

用于监控客户应收账款的回收情况，评估坏账风险，辅助月末坏账准备计提和年末客户余额函证。

### 生成参数

| 参数 | 说明 |
|------|------|
| 截止日期 | 分析基准日（通常为月末） |
| 汇总维度 | 按客户汇总 或 按发票明细展示 |
| 货币 | 按记账本位币显示，或按原始货币显示 |
| 筛选 | 可选择仅展示逾期账款 |

### 报表内容

**第一部分：发票明细（按客户分组）**

| 字段 | 说明 |
|------|------|
| 客户名称 | 应收款的客户 |
| 发票编号 | ARInvoice 编号 |
| 开票日期 | 发票开具日期 |
| 到期日 | 按信用期限计算的最晚收款日 |
| 发票含税金额 | 发票总金额 |
| 已收款金额 | 已核销的收款累计金额 |
| 未结余额 | 尚未收到的金额 |
| 账龄 | 截止日与到期日的天数差 |
| 账龄区间 | 未到期 / 1-30天 / 31-60天 / … / 1年以上 |

**第二部分：账龄分布汇总（按客户）**

每个客户一行，列出其在各账龄区间的未结余额分布：

| 客户 | 未到期 | 1-30天 | 31-60天 | 61-90天 | 91-180天 | 181-365天 | 1年以上 | 合计 |
|------|--------|--------|---------|---------|---------|----------|---------|------|
| 客户A | … | … | … | … | … | … | … | … |
| 合计 | … | … | … | … | … | … | … | … |

**第三部分：坏账准备测算（辅助计算）**

| 账龄区间 | 未结余额 | 参考损失率 | 应计提准备 |
|---------|---------|-----------|-----------|
| 未到期 | … | 2% | … |
| 逾期 1-30天 | … | 5% | … |
| … | … | … | … |
| **应计坏账准备合计** | | | … |
| **当前坏账准备余额** | | | … |
| **差额（需补提/冲回）** | | | … |

**核对校验：** 报表底部显示"所有客户未结余额合计"与"总账应收账款科目余额"的对比，两者必须一致。

## 10. 应付账款明细与账龄分析表

### 业务用途

用于监控供应商应付账款的到期情况，辅助资金安排和付款计划，同时帮助财务发现逾期未付的账款。

### 生成参数

| 参数 | 说明 |
|------|------|
| 截止日期 | 分析基准日（通常为月末） |
| 汇总维度 | 按供应商汇总 或 按发票明细展示 |
| 货币 | 按记账本位币或原始货币 |
| 筛选 | 可选择仅展示即将到期或已逾期账款 |

### 报表内容

**第一部分：发票明细（按供应商分组）**

| 字段 | 说明 |
|------|------|
| 供应商名称 | 应付款的供应商 |
| 发票编号 | APInvoice 编号 |
| 收票日期 | 收到发票的日期 |
| 到期日 | 按付款条件计算的最晚付款日（高亮显示 7天内到期的） |
| 发票含税金额 | 发票总金额 |

**第二部分：账龄分布汇总（按供应商）**

与应收账款格式相同，行为每个供应商，列为各账龄区间未结余额。

**即将到期提醒：** 报表顶部单独列出"7天内到期未付发票"清单，便于资金安排。

**核对校验：** 报表底部显示"所有供应商未结余额合计"与"总账应付账款科目余额"的对比，两者必须一致。
