# 系统概述

## 系统定位

本系统是一款面向**跨国企业集团**的多公司、多货币会计管理系统，遵循**中国企业会计准则（CAS）**。

系统帮助财务团队完成日常会计记账、财务期间管理、合并报表编制，以及多货币核算等核心财务工作，适用于在多个国家和地区设有公司的集团企业。

---

## 核心理念

### 遵循中国企业会计准则（CAS）

系统设计严格遵循以下准则要求：

| 准则编号 | 准则名称 | 在本系统中的应用 |
|---------|---------|--------------|
| CAS 30 | 财务报表列报 | 资产负债表、利润表、所有者权益变动表列报格式 |
| CAS 31 | 现金流量表 | 直接法与间接法现金流分类 |
| CAS 33 | 合并财务报表 | 母子公司合并、非控制性权益处理 |
| CAS 19 | 外币折算 | 记账本位币确定、汇率折算方法、汇兑差额处理 |
| CAS 14 | 收入 | 收入确认原则 |
| CAS 6  | 无形资产 | 研发支出资本化与费用化判断 |

### 基本会计原则

- **权责发生制**：收入在赚取时确认，费用在发生时确认，与实际收付现金无关
- **复式记账**：每笔交易必须同时记录借方和贷方，且借贷合计必须相等
- **不可撤销性**：已经过账的会计记录不得直接修改，错误只能通过冲销分录更正
- **完整审计追踪**：系统记录所有操作的历史，包括操作人、时间和变更内容

## 目标用户

| 角色 | 主要工作 |
|------|---------|
| 财务总监（CFO） | 审阅合并报表，管控财务期间，授权重大凭证 |
| 财务经理 | 管理科目表和汇率，审批凭证，监控日常记账 |
| 会计 | 录入日常凭证，对账，生成财务报表 |
| 期间管理员 | 管理会计年度和期间开关，控制记账窗口 |
| 合并专员 | 执行月末/年末合并，处理内部交易抵消 |
| 审计员（只读） | 查阅历史凭证、审计日志和财务报表 |

## 系统功能模块

### 1. 组织与公司管理

支持集团企业的多公司管理。一个用户账户可以属于多个组织，每个组织可以包含多家子公司。每家公司独立维护自己的科目表、凭证记录和会计期间。

### 2. 科目表管理

系统采用分级科目体系，遵循 CAS 规定的标准分类（资产、负债、所有者权益、收入、费用），科目编号遵循财政部统一规范。科目支持多级父子层级，可灵活配置公司的账户结构。

### 3. 日记账凭证

所有财务交易通过日记账凭证录入。凭证采用多步审批流程（草稿→提交→审批→过账），确保财务数据的准确性和授权控制。支持多货币凭证录入。

### 4. 会计期间管理

按中国《会计法》规定，会计年度为公历年（1月1日至12月31日），每个会计年度固定包含 **12 个月度期间**。财务人员可灵活开关期间，控制哪个时间窗口允许记账。

### 5. 货币与汇率管理

支持所有 ISO 4217 国际标准货币。汇率按类型（期末汇率、平均汇率、历史汇率）手动录入，用于凭证记录和报表生成。

### 6. 合并报表

支持母子公司合并（完全合并法、权益法、成本法），自动执行货币折算、内部交易抵消和非控制性权益计算，生成集团合并财务报表（依据 CAS 33）。

### 7. 财务报告

依据 CAS 30 生成五大核心财务报表（均为强制编制）：
- **资产负债表**（某一时点的财务状况）
- **利润表**（含营业利润、利润总额等中国特有指标）
- **现金流量表**（某一期间的现金流动情况）
- **所有者权益变动表**（某一期间的权益变动，强制编制）
- **合并财务报表**（含非控制性权益的集团整体报表）

### 8. 权限管理

基于角色的访问控制体系，支持按组织分配不同角色，精细控制每位成员的操作权限。

## 数据隔离与安全

系统采用多租户架构：

- 每个**组织**的数据完全隔离，跨组织访问受到严格限制
- 用户只能访问自己所在组织的数据
- 所有敏感操作均记录在审计日志中，便于事后追溯
- 支持企业单点登录（SSO）和多因素认证
