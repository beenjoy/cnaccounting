# 业务规则

本文档以业务语言描述系统中所有核心业务规则，遵循**中国企业会计准则（CAS）**，供财务团队理解系统的运作逻辑和约束条件。

---

## 1. 复式记账基本原则

### 借贷平衡原则

每张日记账凭证中，所有行的借方金额合计必须等于贷方金额合计。这是复式记账的基本前提，凭证在借贷不平衡的情况下无法提交审批。

对于包含多种货币的凭证，每种货币下的借贷必须分别独立平衡。

### 科目余额方向规则

| 科目类型 | 正常余额方向 | 借方的效果 | 贷方的效果 |
|---------|------------|----------|----------|
| 资产 | 借方 | 增加 | 减少 |
| 费用 | 借方 | 增加 | 减少 |
| 负债 | 贷方 | 减少 | 增加 |
| 所有者权益 | 贷方 | 减少 | 增加 |
| 收入 | 贷方 | 减少 | 增加 |

### 权责发生制

- **收入确认**：收入在赚取时确认，与是否已收到现金无关（依据 CAS 14《收入》）
- **费用确认**：费用在发生时确认，与是否已支付现金无关
- 期末通过调整凭证处理应计项目（已发生但尚未记录）和递延项目（已记录但尚未赚取/发生）

### 已过账凭证的不可修改性

凭证一旦完成过账，其内容永久锁定，不可修改也不可删除。如需纠错，必须新建一张冲销凭证来反向抵消原凭证的影响，再录入正确内容。

### 凭证过账的原子性

一张凭证中的所有行必须同时过账。过账要么全部成功，要么全部失败，不存在部分过账的情况。

## 2. 凭证审批流程规则

### 凭证状态流转

凭证从创建到最终入账，经历以下状态：

**草稿 → 待审批 → 已审批 → 已过账（→ 已冲销）**

- 凭证创建后处于**草稿**状态，可自由编辑
- 提交后进入**待审批**状态，不可再编辑
- 审批通过进入**已审批**状态，等待执行过账
- 过账后进入**已过账**状态，数据永久锁定
- 审批不通过则退回**草稿**状态，须附上拒绝原因

### 各操作所需权限

| 操作 | 操作前的状态 | 操作后的状态 | 备注 |
|------|------------|------------|------|
| 提交审批 | 草稿 | 待审批 | 需要提交权限 |
| 审批通过 | 待审批 | 已审批 | 需要审批权限 |
| 拒绝退回 | 待审批 | 草稿 | 需要审批权限，须填写拒绝原因 |
| 过账 | 已审批 | 已过账 | 需要过账权限 |
| 冲销 | 已过账 | 已冲销 | 需要冲销权限 |
| 删除 | 草稿 | — | 仅创建者或管理员可删除草稿凭证 |

### 过账前的五项验证

凭证在过账时，系统将逐一检验以下五个条件，全部通过才能完成过账：

1. **借贷平衡**：所有凭证行的借方合计等于贷方合计（以记账本位币计算）
2. **科目有效**：凭证中引用的所有科目必须处于有效状态且允许直接记账
3. **期间开放**：凭证交易日期对应的会计期间必须处于开放状态
4. **货币一致性**：同一张凭证中，相同货币的行必须使用相同的汇率
5. **内部往来匹配**：如涉及集团内部往来科目，对应公司的账簿中必须有匹配的凭证行

### 冲销凭证规则

- 只有处于**已过账**状态的凭证才可以被冲销
- 已经冲销过的凭证不可再次冲销
- 冲销凭证是将原凭证所有行的借贷方向互换后形成的新凭证
- 冲销凭证与原凭证相互关联，可双向追溯
- 冲销凭证创建后处于**草稿**状态，需独立走完完整的审批和过账流程
- 若原凭证涉及内部往来，对方公司的对应凭证也应同步冲销

### 凭证编号规则

- 凭证编号格式为：年份 + 顺序号（如 JE-2025-00001）
- 编号**仅在过账时**由系统自动分配，按公司内顺序递增
- 草稿和待审批状态的凭证没有编号
- 过账后编号永久绑定，不可更改

### 凭证只读删除规则

- 只有**草稿**状态的凭证可以删除
- 已过账和已冲销的凭证永远不会从系统中删除（只追加，不删除）

## 3. 会计期间管理规则

### 12期间月度结构

按中国《会计法》规定，会计年度为公历年（1月1日至12月31日），每个会计年度必须包含恰好 **12 个月度期间**：

| 期间编号 | 类型 | 日期范围 | 主要用途 |
|---------|------|---------|---------|
| 第 1–11 期 | 月度期间 | 各月第一天至月末 | 日常业务记账 |
| 第 12 期 | 12月期间 | 12月1日至12月31日 | 日常业务记账及年末调整分录 |

**说明：** 年末审计调整分录、税务调整分录、结账分录等均在第12期（12月）录入和过账，无单独的"调整期"。

### 期间状态规则

每个期间只有两种状态：**开放**和**关闭**。

| 操作 | 前置状态 | 结果状态 | 额外要求 |
|------|---------|---------|---------|
| 关闭期间 | 开放 | 关闭 | 需要期间管理权限 |
| 重新开放期间 | 关闭 | 开放 | 需要期间管理权限，且**必须填写重开原因**（不少于10个字符）；每次重开操作均记录在审计日志中 |

- 期间处于**开放**状态时，允许向该期间过账凭证
- 期间处于**关闭**状态时，禁止新增凭证，也不允许修改已有记录

### 会计年度状态规则

会计年度有两种状态：**开放**和**已关闭**。

- 关闭会计年度前，其全部 12 个期间必须全部处于关闭状态
- 同一公司的不同会计年度日期不可重叠
- 关闭后的会计年度若需重新调整，需特殊授权并填写原因

### 月末关闭前建议核查清单

在关闭某一期间前，财务人员应完成以下核查（系统提示提醒，但不强制阻止关闭）：

| 核查项目 | 说明 |
|---------|------|
| 银行余额核对 | 账面现金余额与银行对账单余额核对一致 |
| 应收账款核对 | 应收账款明细汇总与总账余额一致 |
| 应付账款核对 | 应付账款明细汇总与总账余额一致 |
| 存货估值 | 期末存货已按正确方法估值 |
| 固定资产折旧 | 本期折旧分录已录入并过账 |
| 预付费用摊销 | 预付费用本期摊销分录已处理 |
| 应计项目审查 | 已发生但尚未入账的收入和费用均已通过调整凭证确认 |
| 内部往来核对 | 集团内部公司间往来差异已查明并处理 |
| 外币重估 | 外币货币性资产和负债已按期末汇率重估 |
| 试算平衡检查 | 期末试算平衡借贷相等 |
| 财务主管审批 | 财务负责人完成最终审核确认 |

## 4. 年末结账流程规则

### 前提条件

执行年末结账前，以下条件必须全部满足：
- 公司已指定未分配利润科目
- 该会计年度全部 12 个期间均已关闭
- 该年度尚未执行过年末结账

### 两步结账流程

**第一步：启动结账（Begin Close）**

系统自动生成结账凭证，过入第12期（12月）：
- 将所有收入类科目的余额转入未分配利润（使收入科目余额归零）
- 将所有费用类科目的余额从未分配利润中扣除（使费用科目余额归零）
- 净效果：当年净利润（收入合计 - 费用合计）转入未分配利润

**第二步：完成结账（Complete Close）**

确认结账分录无误后，正式完成年末结账，将年度状态标记为已关闭，结账分录同时自动过账。

### 盈余公积计提规则（CAS 特有要求）

依据中国公司法规定，年末结账完成后，企业须从当年净利润中提取**法定盈余公积**：

| 规则 | 说明 |
|------|------|
| 提取比例 | 净利润的 **10%** |
| 停止提取条件 | 法定盈余公积累计金额达到**注册资本的 50%** 后，可不再提取 |
| 会计处理 | 借记"未分配利润"，贷记"盈余公积（法定盈余公积）" |
| 顺序 | 提取盈余公积**先于**向股东分配利润 |

- 盈余公积不得随意动用，用途受法律限制（如弥补亏损、转增资本等）
- 盈余公积的提取应作为结账流程的一部分，在年末结账完成后由财务经理手动发起

### 约束规则

- 每个会计年度只能执行一次年末结账
- 结账后若需调整，须经特殊授权（重开年度），调整完毕后再次执行结账流程
- 结账分录由系统自动生成，财务人员不可手动删除

## 5. 多货币处理规则（依据 CAS 19《外币折算》）

### 记账本位币确定

- 每家公司的记账本位币在创建时确定，确定后不可随意更改
- 记账本位币是该公司日常经营活动产生和使用现金流最主要的货币
- 中国境内注册的企业，记账本位币通常为人民币（CNY）
- 所有账簿余额均以记账本位币为基准计量

### 折算规则（记账本位币 → 报告货币）

当需要将某家公司的数据换算为集团报告货币时，遵循以下规则：

| 报表项目 | 使用的汇率 | 说明 |
|---------|---------|------|
| 资产（全部） | 期末汇率 | 报表日当天的市场汇率 |
| 负债（全部） | 期末汇率 | 报表日当天的市场汇率 |
| 实收资本/股本、资本公积 | 历史汇率 | 各次股权发行时的历史汇率 |
| 盈余公积、未分配利润 | 计算推导 | 不直接折算，由历史净利润累加推算 |
| 收入、费用 | 平均汇率 | 当期期间的加权平均汇率 |
| 折算差额 | 差额法 | 使资产 = 负债 + 所有者权益，差额记入其他综合收益 |

### 汇兑差额处理（CAS 19 规定）

CAS 19 对不同类型的汇兑差额采用不同处理方式：

| 差额类型 | 处理方式 | 说明 |
|---------|---------|------|
| 货币性项目汇兑差额 | 计入**财务费用**（损益） | 外币现金、应收账款、应付账款等按期末汇率重估产生的差额 |
| 非货币性外币项目 | 维持历史汇率，不重估 | 存货、固定资产等非货币性项目不进行期末重估 |
| 境外子公司报表折算差额 | 计入**其他综合收益** | 将境外子公司数据折算为报告货币时产生的整体折算差额 |

境外子公司处置时，历史积累在其他综合收益中的折算差额才最终转入当期损益（投资收益）。

### 期末外币重估

期末需对以外币计价的货币性资产和负债（如外币现金、外币应收账款、外币应付账款等）按期末汇率重新计量：
- 重估后的记账本位币金额与原账面金额的差额，即为**汇兑差额**，计入当期**财务费用**
- 非货币性项目（如存货、固定资产）不进行重估，维持历史汇率不变
- 每个期间只能执行一次重估，重跑须先撤销原重估记录

### 汇率查找规则

系统严格按照（来源货币、目标货币、生效日期、汇率类型）四要素精确匹配汇率。若找不到对应汇率，系统将拒绝操作并提示错误，不会自动使用最近日期的汇率代替。

## 6. 合并报表规则（依据 CAS 33《合并财务报表》）

### 合并方法的选择

根据母公司对被投资公司的控制程度，采用不同的合并处理方式：

| 持股比例 / 控制情况 | 合并方法 | 处理方式 |
|------------------|---------|---------|
| 通常超过50%投票权，或通过其他方式取得控制权 | 完全合并法 | 将子公司所有资产、负债、收入、费用100%纳入合并，同时计算并列示非控制性权益 |
| 通常20%–50%，有重大影响但无控制权 | 权益法 | 以单行"长期股权投资"科目列示，按持股比例确认被投资公司的损益 |
| 通常低于20%，既无控制也无重大影响 | 成本法 | 以初始投资成本计量，仅在收到股息时确认投资收益 |

### 合并执行的前置条件

以下条件必须全部满足，才能启动合并执行：
- 所有参与合并的成员公司均已关闭本次合并目标期间
- 所有涉及的货币折算汇率已经录入系统
- 所有成员公司的试算表处于平衡状态
- 同一合并组、同一期间，当天内不可重复运行合并

### 内部交易抵消规则

在编制合并报表时，集团内部交易必须完全消除：

| 抵消类型 | 处理说明 |
|---------|---------|
| 内部应收/应付 | 一方对另一方的应收款与对方的应付款完全相互抵消，两端金额必须一致 |
| 内部收入/费用 | 销售方确认的内部收入与购买方确认的采购成本同时消除 |
| 存货中未实现利润 | 消除集团内部销售利润中，买方期末还持有的那部分存货所含的利润 |
| 固定资产未实现利润 | 消除集团内部资产转让产生的利润，并调整后续折旧 |
| 内部股息 | 消除母子公司之间的股息收入与股息支付 |
| 内部投资 | 消除母公司对子公司的长期股权投资与子公司的所有者权益（实收资本）对应关系 |

### 非控制性权益计算规则

- 非控制性权益在资产负债表所有者权益部分**单独列示**（不与母公司权益合并）
- 非控制性股东在利润中的份额在利润表净利润下方**单独列示**
- 计算方法：子公司净资产 × 非控制性权益持股比例 = 非控制性权益额

### 合并的关键约束

- 合并执行分 8 个步骤，必须按顺序完成，不可跳过任何步骤
- 所有抵消分录均过入合并目标期间
- 合并一旦开始，不可在执行过程中修改成员公司数据

## 7. 科目层级结构规则

- 父科目必须属于同一家公司（不允许跨公司层级）
- 父科目和子科目的科目类型必须完全一致（如父科目是资产类，子科目也必须是资产类）
- 不允许形成循环层级（如 A 是 B 的父科目，B 是 A 的父科目）
- 科目层级深度没有限制，可以根据业务需要设置多层
- 父（汇总）科目通常不允许直接用于记账，仅用于汇总
- 若要停用父科目，必须先停用其所有子科目
- 有余额的科目不可停用

---

## 8. 汇率管理规则

- 来源货币和目标货币不可相同（不允许录入 CNY→CNY 这样的汇率）
- 汇率值必须为正数
- 同一货币对、同一日期、同一类型的汇率记录在系统中唯一，不可重复录入
- 已录入的汇率不可修改，如需更正须删除后重新录入（以保持审计轨迹）

---

## 9. 数据安全与隔离规则

- 每个组织的数据完全隔离，不同组织之间无法互相访问数据
- 用户只能访问自己所属组织的数据
- 所有操作均需要经过身份验证，访问权限基于在该组织中的角色严格控制

---

## 10. 系统内置保护规则

系统内置以下四条保护规则，无法被删除或绕过：

| 规则名称 | 效果 | 说明 |
|---------|------|------|
| 已关闭期间保护 | 禁止 | 已关闭的会计期间不允许过账任何凭证 |
| 有效科目要求 | 禁止 | 已停用的科目不可用于新凭证记账 |
| 借贷平衡验证 | 禁止 | 借贷不平衡的凭证不可过账 |
| 所有者不可移除 | 禁止 | 拥有者（Owner）角色的成员不可被非拥有者移除 |

## 11. 审计追踪要求

以下所有操作均须自动记录在审计日志中，且审计日志只读，不可修改：

| 业务领域 | 必须记录的操作 |
|---------|-------------|
| 日记账凭证 | 创建、提交、审批通过、审批拒绝、过账、冲销 |
| 会计期间 | 开放、关闭、重新开放（含重开原因） |
| 会计年度 | 启动年末结账、完成年末结账、重开年度 |
| 会计科目 | 创建、修改、停用、重新启用 |
| 组织成员 | 发送邀请、接受邀请、角色变更、停用、恢复、移除 |
| 权限规则 | 创建、修改、删除 |
| 组织与公司 | 创建、修改关键信息 |
| 外币重估 | 执行完成（含使用的汇率和产生的汇兑差额金额） |
| 合并执行 | 开始、完成、失败 |

每条审计记录包含：操作人、操作时间、操作类型、操作对象，以及每个被修改字段的变更前和变更后的值。

## 12. 增值税管理规则

### 12.1 税率适用规则

| 纳税人类型 | 适用税率 | 说明 |
|-----------|---------|------|
| 一般纳税人 | 13% | 销售货物、提供加工修理修配劳务 |
| 一般纳税人 | 9% | 农产品、不动产、建筑服务、交通运输 |
| 一般纳税人 | 6% | 现代服务业、生活服务业、金融服务 |
| 一般纳税人 | 0% | 出口货物及服务（适用免税或退税） |
| 小规模纳税人 | 3% | 统一征收率（不区分进销项） |

每家公司在 VATConfig 中配置其纳税人类型；每笔交易依据业务类型选择适用税率。

### 12.2 进项税额抵扣规则（一般纳税人适用）

以下**四个条件须同时满足**，进项税额方可抵扣：

1. **取得合规发票**：收到增值税专用发票（包括电子专票）
2. **完成认证**：在税务系统中成功认证该发票
3. **实际发生业务**：真实发生采购行为，并有合同、入库单等佐证
4. **用于应税项目**：采购的货物或服务用于产生增值税应税收入

**不可抵扣的进项税额（即使取得专票）：**
- 用于简易计税方法的项目
- 用于免税项目
- 用于集体福利或个人消费
- 非正常损失的购进货物
- 购进的旅客运输服务、贷款服务、餐饮服务（用于个人消费时）

### 12.3 月末增值税结算规则

**步骤一：计算当月应纳税额**

```
应纳税额 = 当月销项税额合计 - 当月可抵扣进项税额合计
```

**步骤二：根据计算结果处理**

| 计算结果 | 处理方式 |
|---------|---------|
| 应纳税额 > 0 | 将应纳税额从"应交增值税"转入"未交增值税"，次月申报期内缴纳 |
| 应纳税额 < 0 | 产生留抵税额，结转至下期继续抵扣，无需缴税 |
| 应纳税额 = 0 | 无需申报缴税，当月进销项税额恰好相等 |

**步骤三：计算附加税（以实际缴纳增值税为计税基数）**

| 附加税种 | 计税基数 | 标准税率 |
|---------|---------|---------|
| 城市维护建设税 | 实际缴纳增值税 | 7%（市区）/ 5%（县城）/ 1%（其他） |
| 教育费附加 | 实际缴纳增值税 | 3% |
| 地方教育附加 | 实际缴纳增值税 | 2% |

**注：** 附加税仅在实际缴纳增值税时才产生，留抵期间不计提附加税。

### 12.4 小规模纳税人计算规则

小规模纳税人不区分进项和销项，直接按含税销售额换算：

```
应纳增值税 = 含税销售额 ÷ (1 + 3%) × 3%
```

小规模纳税人取得的进项发票**不可抵扣**，全额计入成本或费用。

## 13. 应收/应付账款管理规则

### 13.1 子账本与总账一致性约束

**这是最核心的 AR/AP 完整性要求：**

- 所有客户应收账款子账本余额之和，必须始终等于总账"应收账款"科目的余额
- 所有供应商应付账款子账本余额之和，必须始终等于总账"应付账款"科目的余额
- 任何破坏此等式的操作，系统必须拒绝执行

### 13.2 发票核销规则

| 规则 | 说明 |
|------|------|
| 多次收款 | 一张发票可通过多次收款逐步核销（部分核销） |
| 批量核销 | 一笔收款可同时核销多张发票 |
| 金额限制 | 单次核销金额不得超过发票的当前未结余额 |
| 不可负数 | 核销后发票未结余额不可为负数 |
| 状态联动 | 核销后系统自动更新发票核销状态（未核销→部分核销→已核销） |

### 13.3 账龄计算规则

账龄以**发票到期日**（非开票日）为起点，与当前日期比较：

| 账龄区间 | 判断逻辑 |
|---------|---------|
| 未到期 | 到期日 ≥ 今天 |
| 逾期 1–30 天 | 今天 - 到期日 ∈ [1, 30] |
| 逾期 31–60 天 | 今天 - 到期日 ∈ [31, 60] |
| 逾期 61–90 天 | 今天 - 到期日 ∈ [61, 90] |
| 逾期 91–180 天 | 今天 - 到期日 ∈ [91, 180] |
| 逾期 181–365 天 | 今天 - 到期日 ∈ [181, 365] |
| 逾期 1 年以上 | 今天 - 到期日 > 365 |

账龄分析以**发票未结余额**参与计算，已全额核销的发票不参与账龄统计。

### 13.4 坏账准备计提规则（CAS 22 预期信用损失模型）

**方法一：账龄分析法（最常用）**

按账龄区间设定损失率，计提坏账准备：

| 账龄区间 | 参考损失率 |
|---------|-----------|
| 未到期 | 1%–3% |
| 逾期 1–30 天 | 5% |
| 逾期 31–90 天 | 10%–20% |
| 逾期 91–180 天 | 30%–50% |
| 逾期 181–365 天 | 50%–80% |
| 逾期 1 年以上 | 80%–100% |

**注：** 具体损失率应由企业根据历史坏账数据和行业特点自行制定，以上为参考范围。

**方法二：个别认定法**

对于金额重大的单项应收款，逐笔评估可回收性，独立计提坏账准备（不参与账龄分析法汇总计算）。

**坏账准备账务规则：**
- 坏账准备科目（1221）是应收账款（1122）的备抵科目
- 期末坏账准备余额 = 各账龄区间未结余额 × 对应损失率 之和
- 若当前准备余额不足，补提差额；若计提过多，冲回差额
- 实际发生坏账时：借坏账准备，贷应收账款（核销）

### 13.5 暂估应付款规则

**适用场景：** 月末，货物或服务已实际收到，但供应商发票尚未到达。

**处理要求：**
1. 月末按暂估金额入账（不含税，因无发票不可确认进项税）
2. 计入专设的"应付账款—暂估"科目，与正式应付账款分开管理
3. 次月收到正式发票后，先冲回暂估分录，再录入正式发票凭证（含进项税）
4. 暂估入账不生成 APInvoice 记录，收到发票后才录入 APInvoice

## 14. 固定资产管理规则

### 14.1 资本化标准

固定资产须同时满足以下条件方可资本化（计入固定资产科目）：

1. 与该资产有关的经济利益很可能流入企业
2. 该资产的成本能够可靠地计量
3. 为生产商品、提供劳务、出租或经营管理而持有
4. 预计使用寿命超过一个会计年度

企业应自行制定并公开披露资本化金额标准（如：单价超过 5,000 元），低于该标准的小额资产可一次性计入当期费用。一旦确定，该标准须保持一致，不得随意变更。

### 14.2 折旧计提规则

| 规则 | 说明 |
|------|------|
| 开始时间 | 固定资产**取得次月**起开始计提折旧 |
| 停止时间 | 资产处置或报废**当月**停止计提折旧 |
| 停用资产 | 停用期间**仍须继续**计提折旧 |
| 已减值资产 | 减值确认后，以减值后账面净值重新计算剩余期间的月折旧额 |
| 已提足折旧 | 账面净值等于预计净残值后，停止计提折旧 |
| 方法一致性 | 折旧方法一经确定，不得随意变更（变更属于会计政策变更，须披露） |

**四种折旧方法规则：**

| 方法 | 适用场景 | 特点 |
|------|---------|------|
| 直线法（年限平均法） | 各期使用程度相近的资产 | 每月折旧额固定，计算简单 |
| 双倍余额递减法 | 前期使用效率高的资产（如电子设备） | 前期折旧多、后期少；最后两年改用直线法 |
| 年数总和法 | 前期使用效率高的资产 | 前期折旧多、后期少 |
| 工作量法 | 使用程度差异大的资产（如车辆、矿山机械） | 按实际工作量计算，与使用量挂钩 |

各方法具体计算公式详见 11-fixed-assets.md。

### 14.3 后续支出资本化规则

**计入当期费用（收益性支出）：**
- 日常维修和保养支出，不延长资产使用年限
- 不提高资产性能或价值

**计入固定资产原值（资本性支出）：**
- 改建、扩建延长使用寿命的支出
- 改良提升资产性能或使资产超过原设计能力的支出
- 重大技术改造支出

**判断原则：** 若后续支出导致资产可带来的未来经济利益超过原先估计的标准，则应资本化。

### 14.4 资产处置规则

处置固定资产时，须按顺序执行以下步骤：

1. 结转至处置当月的折旧（若已在月初处置，则不再计提当月折旧）
2. 将资产原值和累计折旧转出至"固定资产清理"（1606）科目
3. 确认处置收入（若有）
4. 计算并确认处置损益（计入"资产处置收益"6131 或"营业外支出"6301）
5. 资产状态更新为"已处置"

### 14.5 资产减值规则（CAS 8）

**减值迹象检查时机：** 每年年末，以及发生以下迹象时应及时检查：
- 资产市价大幅下跌，且预计不会恢复
- 技术、市场等环境重大不利变化
- 市场利率上升导致折现率提高
- 资产发生严重损坏或停产长期闲置
- 资产连续亏损且预期无改善

**可收回金额确定：**
```
可收回金额 = max（公允价值减处置费用，预计未来现金流量现值）
```

**减值损失确认：**
- 账面净值 > 可收回金额 → 确认减值损失，差额计入"资产减值损失"（6701）
- 减值准备计入固定资产减值准备（1603，备抵科目）

**不可转回原则：** 依据 CAS 8，固定资产减值损失一旦确认，**在后续会计期间不得转回**（与金融资产不同）。
