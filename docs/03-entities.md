# 业务实体

本文档描述系统中所有核心业务实体的含义、包含的信息以及彼此之间的关系。所有实体以业务视角描述，遵循**中国企业会计准则（CAS）**，供财务团队理解系统的数据结构。

---

## 1. 组织（Organization）

**是什么：** 系统中的最高管理容器，代表一个集团企业整体。一个用户账户可以同时属于多个组织，各组织的数据彼此完全隔离。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 名称 | 组织的显示名称 |
| 报告货币 | 编制合并报表所使用的统一货币（如人民币、美元） |
| 时区 | 组织所在时区，影响日期显示 |
| 创建时间 | 组织创建的时间 |

**业务关系：**
- 一个组织包含多家**公司**
- 一个组织可有多个**成员**，每位成员在组织内拥有各自的角色
- 一个组织可定义多个**合并组**
- 一个组织可维护**集团科目表**

**业务规则：**
- 报告货币一旦设定，变更需谨慎（影响合并报表历史数据）
- 仅当组织内没有任何公司时，才可删除该组织

---

## 2. 公司（Company）

**是什么：** 组织下的一个独立法人实体，拥有独立的科目表、独立的凭证记录和独立的会计期间。公司是日常会计记账的基本单元。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 名称 / 法定名称 | 公司显示名称和法律注册名称 |
| 注册国家/地区 | 公司注册所在国（影响合规要求） |
| 税务登记号 | 公司税务识别号（可选） |
| 注册日期 | 公司成立/注册日期（可选） |
| 公司类型 | 股份公司、有限责任公司、合伙企业、独资企业、非营利组织、合作社、分支机构等 |
| 记账本位币 | 公司主要经营活动所使用的货币（依据 CAS 19 确定，中国境内企业通常为人民币） |
| 报告货币 | 公司对外报告所用货币（可与记账本位币不同） |
| 财政年度结束日 | 按中国《会计法》规定，固定为每年12月31日（公历年） |
| 未分配利润科目 | 年末结账时净利润转入的指定科目 |
| 是否有效 | 标记公司是否仍在使用（停用不等于删除） |

**业务关系：**
- 属于一个**组织**
- 拥有独立的**科目表**（一组会计科目）
- 拥有独立的**会计年度**和**会计期间**
- 产生独立的**日记账凭证**
- 可参与一个或多个**合并组**

**业务规则：**
- 记账本位币由公司主要经营活动所在地的货币决定（中国境内企业通常为人民币 CNY）
- 存在会计年度记录或合并关系时，公司不可删除，只能停用
- 每家公司有且只有一个指定的未分配利润科目

---

## 3. 会计科目（Chart Account）

**是什么：** 公司科目表中的一个具体科目，用于对所有财务交易进行分类记录。每个科目都有固定的类型，决定它在报表中的位置。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 科目编号 | 公司内唯一的数字编号，遵循财政部统一规定的编码体系 |
| 科目名称 | 科目的业务描述名称 |
| 科目说明 | 详细的使用说明（可选） |
| 科目类型 | 资产、负债、所有者权益、收入、费用五类之一 |
| 科目类别 | 类型下的细化分类（如资产类下的流动资产、非流动资产） |
| 正常余额方向 | 借方余额正常（资产、费用）或贷方余额正常（负债、所有者权益、收入） |
| 父科目 | 所属的上级汇总科目（可选，用于多级层级结构） |
| 层级深度 | 在科目树中的位置（第1层为顶级） |
| 是否可过账 | 是否允许在此科目直接录入凭证（汇总科目通常不可过账） |
| 现金流类别 | 经营活动、投资活动、融资活动，用于现金流量表分类 |
| 是否内部往来科目 | 该科目是否专门用于集团内部公司间交易 |
| 关联内部往来公司 | 若为内部往来科目，记录对应的关联公司 |
| 货币限制 | 若指定，则该科目只允许以特定货币记账 |
| 是否未分配利润科目 | 标记为年末结账专用的未分配利润科目（每家公司仅一个） |
| 是否有效 | 科目是否仍在使用中 |
| **——增值税关联——** | |
| 是否增值税专用科目 | 标记此科目是否专门用于记录增值税（如应交增值税2221及其子科目）；默认否 |
| 增值税科目用途 | 仅当"是否增值税专用科目=是"时有效：进项税额 / 销项税额 / 未交增值税 / 应交增值税（汇总） |
| **——AR/AP 子账本关联——** | |
| 是否AR子账本明细科目 | 标记此科目是否为某客户的应收账款明细科目（如1122-C00001）；默认否 |
| 关联客户 | 若为AR明细科目，关联的客户档案（Customer）；确保子账本余额可精确追溯到客户 |
| 是否AP子账本明细科目 | 标记此科目是否为某供应商的应付账款明细科目（如2202-V00001）；默认否 |
| 关联供应商 | 若为AP明细科目，关联的供应商档案（Vendor）；确保子账本余额可精确追溯到供应商 |
| **——固定资产关联——** | |
| 是否固定资产专用科目 | 标记此科目是否与某固定资产绑定（资产原值账户、累计折旧账户或折旧费用账户）；默认否 |
| 固定资产科目用途 | 仅当"是否固定资产专用科目=是"时有效：资产原值 / 累计折旧（备抵）/ 折旧费用 |
| 关联固定资产 | 若为固定资产专用科目，关联的固定资产卡片（FixedAsset）（可选，资产建档时绑定） |

**科目编号规范（遵循财政部统一科目编码体系）：**

| 编号段 | 科目大类 | 典型科目 |
|--------|---------|---------|
| 1001–1999 | 资产类 | 库存现金（1001）、银行存款（1002）、应收账款（1122）、存货（1401）、固定资产（1601）、无形资产（1701） |
| 2001–2999 | 负债类 | 短期借款（2001）、应付账款（2202）、应交税费（2221）、长期借款（2501）、递延所得税负债（2511） |
| 3001–3999 | 所有者权益类 | 实收资本/股本（3001）、资本公积（3101）、盈余公积（3103）、未分配利润（3141）、其他综合收益（3121） |
| 4001–4999 | 成本类 | 生产成本（4101）、制造费用（4105） |
| 5001–5999 | 损益类（收入） | 主营业务收入（5001）、其他业务收入（5051）、营业外收入（5301） |
| 6001–6999 | 损益类（费用） | 主营业务成本（6001）、税金及附加（6051）、销售费用（6601）、管理费用（6602）、财务费用（6603）、研发费用（6604）、营业外支出（6711） |

**科目类别详细列表：**

| 类型 | 类别名称 |
|------|---------|
| 资产 | 流动资产、非流动资产、固定资产、无形资产 |
| 负债 | 流动负债、非流动负债 |
| 所有者权益 | 实收资本/股本、资本公积、盈余公积（法定盈余公积）、未分配利润、其他综合收益 |
| 收入 | 主营业务收入、其他业务收入、营业外收入 |
| 费用 | 主营业务成本、税金及附加、销售费用、管理费用、研发费用、财务费用、营业外支出 |

**业务关系：**
- 属于一家**公司**
- 可有一个**父科目**（同公司内，且科目类型必须相同）
- 可有多个**子科目**
- 被**凭证行**引用用于记账
- AR明细科目 → 关联一个**客户档案**（Customer），实现应收账款子账本
- AP明细科目 → 关联一个**供应商档案**（Vendor），实现应付账款子账本
- 固定资产专用科目 → 关联一件**固定资产卡片**（FixedAsset），实现资产账面价值追踪

**业务规则：**
- 同一公司内科目编号不得重复
- 科目一旦创建，其类型（资产/负债/所有者权益/收入/费用）不可修改
- 有余额的科目不可删除或停用
- 父科目的类型必须与子科目一致（不允许跨类型建立层级）
- 汇总科目（有子科目的科目）通常不允许直接记账
- 同一固定资产卡片最多关联三个专用科目（资产原值科目、累计折旧科目、折旧费用科目各一个）
- AR/AP明细科目只能绑定一个客户/供应商；同一客户/供应商只能绑定一个明细科目

## 4. 日记账凭证（Journal Entry）

**是什么：** 记录一笔经济业务的会计文件，是系统中所有财务数据的录入入口。每张凭证包含至少两行借贷记录，且借贷合计必须相等。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 凭证编号 | 系统过账时自动分配的流水号（如 JE-2025-00001） |
| 参考号 | 外部业务单据编号（如发票号、合同号） |
| 说明 | 描述该笔交易业务背景的文字（必填） |
| 交易日期 | 经济业务实际发生的日期 |
| 过账日期 | 凭证正式入账的日期（过账后自动记录） |
| 单据日期 | 原始业务单据的日期（如发票日期，可选） |
| 所属会计年度与期间 | 该凭证归属的会计年度和期间编号（1–12） |
| 凭证类型 | 普通凭证、调整凭证、结账凭证、期初凭证、冲销凭证、周期性凭证、内部往来凭证、重估凭证、抵消凭证、系统凭证 |
| 来源模块 | 凭证来源（总账手工录入、应收模块、应付模块、固定资产模块、增值税模块、合并模块、外币重估等） |
| 来源单据 | 关联的具体业务单据（选填，与来源模块配套使用）：来源=应收模块时关联销售发票（ARInvoice）；来源=应付模块时关联采购发票（APInvoice）；来源=固定资产模块时关联资产卡片（FixedAsset）；来源=增值税模块时关联月度申报记录（VATDeclaration） |
| 是否多货币 | 凭证中是否包含多种货币 |
| 凭证状态 | 当前所处流程阶段（见下方状态说明） |
| 是否为冲销凭证 | 标记该凭证是否是对另一张凭证的冲销 |
| 创建人 / 过账人 | 录入凭证和执行过账操作的用户 |

**凭证状态流程：**

| 状态 | 说明 |
|------|------|
| 草稿 | 刚创建，可自由编辑，尚未进入审批 |
| 待审批 | 已提交，等待有权限的人员审批，不可编辑 |
| 已审批 | 审批通过，等待执行过账操作 |
| 已过账 | 正式入账，数据锁定，不可修改 |
| 已冲销 | 该凭证已被冲销凭证反向抵消 |

**业务关系：**
- 属于一家**公司**
- 包含多条**凭证行**
- 归属于一个**会计期间**
- 可与另一张凭证形成冲销关系
- 可由一张**销售发票**（ARInvoice）触发生成（来源=应收模块时）
- 可由一张**采购发票**（APInvoice）触发生成（来源=应付模块时）
- 可由一件**固定资产**（FixedAsset）的折旧/处置触发生成（来源=固定资产模块时）
- 可由**增值税申报**（VATDeclaration）的缴税操作触发生成（来源=增值税模块时）

**业务规则：**
- 凭证借贷合计必须相等，否则无法提交
- 过账后不可修改，只能通过新建冲销凭证来纠错
- 向已关闭期间过账会被系统拒绝
- 多货币凭证中，每种货币的借贷必须分别平衡

## 5. 凭证行（Journal Entry Line）

**是什么：** 日记账凭证中的单条借贷记录，指定了受影响的科目及金额。每张凭证至少有两行，一借一贷。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 行序号 | 在凭证中的排列顺序 |
| 科目 | 该行记入的会计科目 |
| 借方金额 | 以交易货币表示的借方金额（与贷方二选一） |
| 贷方金额 | 以交易货币表示的贷方金额（与借方二选一） |
| 记账本位币借方/贷方金额 | 折算为记账本位币后的金额 |
| 汇率 | 交易货币折算为记账本位币所使用的汇率 |
| 行说明 | 对该行的补充描述（可选） |
| 内部往来对应公司 | 若为内部往来交易，注明对方公司 |

**业务规则：**
- 每行只能填借方或贷方，不能同时填写两个方向
- 同一张凭证中，所有行的借方合计必须等于贷方合计
- 记账本位币金额 = 交易金额 × 适用汇率

## 6. 货币（Currency）

**是什么：** 系统支持的国际标准货币，遵循 ISO 4217 货币代码标准。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 货币代码 | 3位国际标准代码（如 CNY、USD、EUR） |
| 货币名称 | 货币全称（如 人民币、US Dollar、Euro） |
| 货币符号 | 显示符号（如 ¥、$、€） |
| 小数位数 | 该货币规定的小数位数（0、2或3位） |
| 是否启用 | 是否在系统中可用 |

**常用货币说明：**

| 货币代码 | 名称 | 小数位 |
|---------|------|--------|
| CNY | 人民币 | 2位 |
| USD | 美元 | 2位 |
| EUR | 欧元 | 2位 |
| GBP | 英镑 | 2位 |
| JPY | 日元 | 0位 |
| HKD | 港元 | 2位 |
| SGD | 新加坡元 | 2位 |
| CHF | 瑞士法郎 | 2位 |

## 7. 汇率（Exchange Rate）

**是什么：** 两种货币之间的换算比率，用于凭证录入时的多货币换算和期末折算。汇率由财务人员手动录入或从外部数据源同步。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 来源货币 | 被换算的货币（如 USD） |
| 目标货币 | 换算成的货币（如 CNY） |
| 汇率值 | 精确汇率数值（高精度，支持多位小数） |
| 生效日期 | 该汇率适用的日期 |
| 汇率类型 | 即期汇率、平均汇率、历史汇率、期末汇率 |
| 数据来源 | 来源标识（如"手动录入"、"欧洲央行"） |
| 录入时间 | 汇率录入系统的时间 |

**汇率类型用途：**

| 类型 | 用途 |
|------|------|
| 期末汇率 | 折算资产负债表中的货币性资产和负债项目 |
| 平均汇率 | 折算利润表中的收入和费用项目 |
| 历史汇率 | 折算所有者权益类项目（如实收资本），保持初始录入汇率不变 |
| 即期汇率 | 凭证录入时使用的当日市场汇率 |

**业务规则（依据 CAS 19《外币折算》）：**
- 来源货币和目标货币不可相同
- 同一货币对、同一日期、同一类型的汇率记录唯一（不可重复录入）
- 合并折算时：货币性资产和负债用期末汇率，收入费用用平均汇率，所有者权益用历史汇率
- 货币性项目（现金、应收账款等）的汇兑差额计入**财务费用**（损益）
- 对境外子公司报表进行货币折算产生的差额计入**其他综合收益**

## 8. 会计年度（Fiscal Year）

**是什么：** 公司用于财务核算和报告的一个完整会计周期。按中国《会计法》规定，会计年度固定为公历年（1月1日至12月31日），每个会计年度包含12个月度期间。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 年度编号 | 会计年度的年份标识（如 2025） |
| 年度名称 | 显示名称（如 2025年度） |
| 开始日期 | 固定为当年1月1日 |
| 结束日期 | 固定为当年12月31日 |
| 状态 | 开放（允许记账）或关闭（年度已结账） |

**业务关系：**
- 属于一家**公司**
- 包含恰好 **12 个月度会计期间**

**业务规则：**
- 每个会计年度固定为公历年（1月1日至12月31日），不可自定义
- 每个会计年度必须恰好包含12个月度期间
- 同一公司的会计年度不可重叠
- 关闭会计年度前，其所有12个期间必须全部处于已关闭状态

---

## 9. 会计期间（Fiscal Period）

**是什么：** 会计年度内的一个具体记账时段。每个会计年度包含12个月度期间（1月至12月），对应公历各月份。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 期间编号 | 1–12，对应1月至12月 |
| 期间名称 | 如"2025年1月"、"2025年12月" |
| 期间类型 | 月度期间（固定） |
| 开始日期 | 期间第一天（如1月1日） |
| 结束日期 | 期间最后一天（如1月31日） |
| 状态 | 开放（可过账）或关闭（不可过账） |
| 关闭人 / 关闭时间 | 执行关闭操作的用户及时间 |
| 增值税申报状态 | 本期间的增值税申报情况（只读，由 VATDeclaration 状态驱动）：未申报 / 已申报 / 已缴税 |

**业务关系：**
- 属于一个**会计年度**
- **凭证**过账时必须归属于一个开放的期间
- 可关联一条**月度增值税申报记录**（VATDeclaration）；申报期间通常与会计期间一一对应（月度申报时）

**业务规则：**
- 开放状态的期间允许过账，关闭后禁止新增凭证
- 重新开放已关闭的期间须提供书面理由，且受权限控制
- 每次重新开放操作均记录在审计日志中
- 年末调整分录（如折旧、摊销、计提等）应在第12期（12月）完成录入和过账
- 增值税申报状态不影响期间开放/关闭状态（两者独立管理）

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 关联期间 | 被重新开放的会计期间 |
| 重开原因 | 必须填写的文字说明（最少10个字符） |
| 操作人 | 执行重开操作的用户 |
| 操作时间 | 重开操作发生的时间 |
| 原期间状态 | 重开前的期间状态（记录为历史） |

## 11. 合并组（Consolidation Group）

**是什么：** 需要合并报表的一组公司的集合，包含一家母公司和若干子公司或关联公司，以及适用于该合并组的抵消规则。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 名称 | 合并组的名称 |
| 报告货币 | 合并报表的统一货币 |
| 默认合并方法 | 该组默认采用的合并方式 |
| 母公司 | 处于控制地位的母公司 |
| 是否有效 | 合并组是否仍在使用 |

**业务关系：**
- 属于一个**组织**
- 包含多个**合并成员**（各子公司/联营公司）
- 关联若干**抵消规则**

## 12. 合并成员（Consolidation Member）

**是什么：** 参与合并的某一家具体子公司或联营公司，记录其在合并组中的持股比例和合并方式。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 关联公司 | 参与合并的那家公司 |
| 持股比例 | 母公司直接或间接持有该公司的股权比例（0%–100%） |
| 合并方法 | 完全合并法、权益法或成本法 |
| 收购日期 | 母公司取得该公司控制权的日期 |
| 商誉金额 | 收购时产生的商誉（可选） |
| 非控制性权益比例 | 不属于母公司的股权比例（= 1 - 持股比例） |

**合并方法适用场景（依据 CAS 33《合并财务报表》）：**

| 合并方法 | 适用条件 | 处理方式 |
|---------|---------|---------|
| 完全合并法 | 母公司持股通常超过50%，拥有控制权 | 将子公司全部资产、负债、收入、费用逐项合并，计算并列示非控制性权益 |
| 权益法 | 持股通常在20%–50%，有重大影响但无控制权 | 按持股比例确认投资收益，以单行"长期股权投资"列示 |
| 成本法 | 持股通常低于20%，无重大影响 | 以初始投资成本计量，仅在收到股息时确认收益 |

## 13. 抵消规则（Elimination Rule）

**是什么：** 定义在合并过程中如何自动识别并抵消特定类型内部交易的规则。规则描述了触发条件、涉及的科目和抵消方向。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 名称 / 说明 | 规则的业务名称和描述 |
| 抵消类型 | 规则所处理的内部交易类型（见下方类型列表） |
| 触发条件 | 满足哪些条件时自动触发此规则 |
| 来源科目 | 参与抵消的科目范围 |
| 目标科目 | 抵消分录写入的科目 |
| 是否自动 | 是否在合并执行时自动生成抵消分录 |
| 优先级 | 多条规则同时适用时的执行顺序（数字越小越先执行） |
| 是否有效 | 规则是否仍在使用 |

**常见抵消类型：**

| 类型 | 业务说明 |
|------|---------|
| 内部应收/应付抵消 | 抵消集团公司间的借贷往来（A公司对B公司的应收款 vs B公司对A公司的应付款） |
| 内部收入/费用抵消 | 抵消集团内部的销售与采购（A公司向B公司销售的收入 vs B公司从A公司采购的成本） |
| 内部股息抵消 | 抵消母子公司之间的股息收支 |
| 内部投资抵消 | 抵消母公司对子公司的长期股权投资与子公司的实收资本 |
| 存货未实现利润 | 消除集团内部销售中尚未向外部出售的存货所含利润 |
| 固定资产未实现利润 | 消除集团内部资产转让产生的未实现利润 |

## 14. 合并执行（Consolidation Run）

**是什么：** 对某一特定会计期间执行合并报表编制操作的完整过程记录，包含执行状态、步骤进展和产生的抵消分录。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 合并组 | 本次执行所属的合并组 |
| 目标年度与期间 | 本次合并针对的会计年度和期间 |
| 截止日期 | 合并数据的基准日（通常为期末日） |
| 执行状态 | 待执行、执行中、已完成、执行失败、已取消 |
| 当前步骤 | 正在进行的处理步骤名称 |
| 抵消凭证清单 | 本次合并自动生成的抵消分录 |
| 验证错误 / 警告 | 数据验证中发现的问题 |
| 发起人 / 完成时间 | 执行合并的操作人及完成时间 |

**合并执行的 8 个步骤（按序执行，不可跳过）：**

| 步骤 | 名称 | 说明 |
|------|------|------|
| 1 | 验证成员数据 | 确认所有参与合并的公司已关闭对应期间，且试算平衡 |
| 2 | 货币折算 | 按 CAS 19《外币折算》将各公司数据折算为合并报告货币 |
| 3 | 汇总余额 | 将所有成员折算后的科目余额加总 |
| 4 | 内部往来匹配 | 识别并核对集团内部公司间往来，找出差异 |
| 5 | 生成抵消分录 | 按优先级依次应用抵消规则，生成消除分录并过账到当期 |
| 6 | 计算非控制性权益 | 按各子公司的非控制性权益比例计算应归属的净利润和净资产份额 |
| 7 | 生成合并试算表 | 汇总得出合并后的完整试算平衡表 |
| 8 | 最终验证 | 验证合并结果的完整性和准确性，生成正式报告 |

## 15. 内部往来交易（Intercompany Transaction）

**是什么：** 记录集团内两家公司之间发生的资金往来或业务交易，用于合并时的匹配核对和抵消处理。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 交易发起方 | 集团内提供商品/服务/资金的一方公司 |
| 交易接收方 | 集团内接收商品/服务/资金的一方公司 |
| 交易类型 | 内部买卖、内部贷款、管理费、股息、资本注入、成本分摊、特许权使用费 |
| 交易日期 | 业务发生日期 |
| 金额 | 交易金额及货币 |
| 匹配状态 | 已匹配、未匹配、部分匹配、差异已审批 |
| 差异金额 | 两方记录金额的差异（如有） |
| 差异说明 | 对差异原因的解释 |

**匹配状态说明：**

| 状态 | 含义 |
|------|------|
| 已匹配 | 双方记录完全一致，可正常抵消 |
| 未匹配 | 一方有记录，对方无对应记录，需核查 |
| 部分匹配 | 双方记录存在金额差异，需确认原因 |
| 差异已审批 | 差异原因已被授权人员确认，允许继续合并 |

## 16. 外币重估执行（Currency Revaluation Run）

**是什么：** 期末将以外币计价的货币性资产和负债按期末汇率重新计量的操作记录。依据 CAS 19，产生的汇兑差额计入当期**财务费用**（损益）。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 关联公司 | 执行重估的公司 |
| 目标年度与期间 | 重估针对的会计期间 |
| 执行日期 | 运行重估的日期 |
| 期末汇率 | 本次重估使用的汇率 |
| 重估方法 | 全资产负债法（对所有货币性项目）或仅未结项目法 |
| 汇兑差额合计 | 本次重估产生的汇兑差额总额（计入财务费用） |
| 生成凭证 | 系统自动生成的重估凭证 |
| 执行人 | 发起重估操作的用户 |

**各科目重估明细包含：**
- 科目名称
- 外币余额
- 重估前记账本位币金额
- 重估后记账本位币金额
- 本科目汇兑差额

**业务规则：**
- 仅对货币性资产和负债项目进行重估（现金、应收账款、应付账款等）
- 非货币性项目（如存货、固定资产）不进行重估，维持历史汇率
- 每个期间只能执行一次重估（如需重跑，须先撤销原重估）
- 汇兑差额计入财务费用（收益为正，损失为负）

## 17. 记账本位币认定记录（Accounting Currency Determination）

**是什么：** 记录某家公司如何依据 CAS 19《外币折算》的规定确定其记账本位币的分析文档，包含评估各项影响因素后得出的结论。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 关联公司 | 接受记账本位币评估的公司 |
| 认定的记账本位币 | 评估后确定采用的记账本位币 |
| 生效日期 | 该记账本位币开始适用的日期 |
| 评估人 / 认定时间 | 进行评估的用户及操作时间 |
| 考量因素列表 | 影响记账本位币判断的各项因素（见下方） |

**CAS 19 记账本位币认定主要考量因素：**

| 因素 | 说明 |
|------|------|
| 销售收入货币 | 商品或服务的销售价格及货款收取主要使用哪种货币 |
| 成本费用货币 | 主要成本、费用的支付主要使用哪种货币 |
| 融资货币 | 公司债务融资主要使用哪种货币 |
| 日常经营资金货币 | 日常收入和留存的资金主要以哪种货币形式存在 |
| 监管要求 | 适用的法规是否明确规定记账本位币（如中国境内注册企业通常以人民币为记账本位币） |

**注：** 中国境内注册的企业，其记账本位币通常为人民币（CNY）。境外注册但在华经营的外商投资企业，也可依据 CAS 19 选择其他货币作为记账本位币，但须报税务机关备案。

**记录范围（涵盖以下实体的操作）：**

**业务规则：**
- 审计日志只可查看，不可修改或删除
- 所有重要操作（不限于凭证过账，包括成员管理、期间开关等）均自动记录
- 查看者角色（审计员）可访问完整的审计日志

## 22. 增值税配置（VATConfig）

**是什么：** 每家公司的增值税基础配置，决定该公司适用的纳税人类型和申报周期。每家公司有且仅有一条配置记录。

| 纳税人类型 | 一般纳税人 或 小规模纳税人 |
| 适用税率 | 该公司使用的增值税税率列表（一般纳税人：13%/9%/6%/0%；小规模：3%） |
| 申报周期 | 月度申报 或 季度申报 |

| 附加税配置 | 城市维护建设税税率（7%/5%/1%）、教育费附加税率（3%）、地方教育附加税率（2%） |

**业务规则：**
- 纳税人类型决定能否取得并抵扣增值税专用发票

## 23. 增值税记录（VATRecord）

**是什么：** 记录每一笔业务交易中涉及的增值税金额，是进项税额和销项税额的明细档案。每张发票对应一条 VATRecord。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 关联日记账凭证 | 产生此税额的会计凭证 |
| 发票类型 | 增值税专用发票 / 增值税普通发票 / 电子专票 / 电子普票 |
| 方向 | 进项（采购端，可抵扣）或 销项（销售端，应缴纳） |
| 税率 | 适用的增值税税率（13%/9%/6%/0% 或 3%） |
| 不含税金额 | 交易的价税分离后金额 |
| 税额 | 本笔交易的增值税额 |
| 含税金额 | 不含税金额 + 税额 |
| 认证状态 | 待认证 / 已认证 / 认证失败（仅进项专票需认证） |
| 认证日期 | 向税务机关完成认证的日期 |
| 所属申报期间 | 该税额计入哪个月度/季度的申报 |
| 是否可抵扣 | 是否满足进项税抵扣的全部条件 |

**业务规则：**
- 进项税额仅在满足以下全部条件时才可抵扣：①取得增值税专用发票、②已完成认证、③实际发生采购业务、④用于应税项目
- 销项税额在开票时即产生纳税义务

## 24. 月度增值税申报（VATDeclaration）

**是什么：** 记录每个申报期的增值税汇总数据和申报状态，对应企业向税务机关提交的增值税申报表。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 所属公司 | 申报主体 |
| 申报期间 | 年月（如 2025-03）或季度（如 2025-Q1） |
| 本期销项税额合计 | 汇总当期所有销项 VATRecord 的税额 |
| 本期进项税额合计 | 汇总当期所有已认证进项 VATRecord 的税额 |
| 应纳税额 | 销项税额 - 进项税额（正数为应缴，负数为留抵） |
| 留抵税额 | 当应纳税额为负数时，结转至下期的税额 |
| 城建税应纳税额 | 实际缴纳增值税 × 适用税率（7%/5%/1%） |
| 教育费附加 | 实际缴纳增值税 × 3% |
| 地方教育附加 | 实际缴纳增值税 × 2% |
| 申报状态 | 草稿 / 已申报 / 已缴税 |
| 申报日期 | 实际提交申报的日期 |
| 缴税日期 | 实际完成缴税的日期 |
| 关联缴税凭证 | 缴纳增值税和附加税的日记账凭证 |

## 应收账款与应付账款子账本实体

## 25. 客户档案（Customer）

**是什么：** 企业的销售客户信息档案，用于管理应收账款子账本、开具增值税发票和信用额度控制。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 客户编号 | 系统内唯一标识（格式：C-XXXXX） |
| 客户名称 | 法定公司名称，与增值税发票抬头一致 |
| 纳税人识别号 | 18位统一社会信用代码（开具专票时必填） |
| 开票地址 | 增值税发票上的注册地址 |
| 开票电话 | 增值税发票上的联系电话 |
| 开户银行及账号 | 增值税发票上填写的银行信息 |
| 信用期限 | 约定付款期限（天数，如 30 天） |
| 信用额度 | 允许赊销的最高未结余额 |
| 应收账款明细科目 | 该客户对应的总账明细科目（如 1122-C00001） |
| 是否有效 | 是否仍在使用 |

**业务规则：**
- 同一组织内，客户编号不可重复
- 开具增值税专用发票时，纳税人识别号为必填项
- 信用额度超限时系统发出警告（不强制阻止）

## 26. 供应商档案（Vendor）

**是什么：** 企业的采购供应商信息档案，用于管理应付账款子账本和进项发票认证。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 供应商编号 | 系统内唯一标识（格式：V-XXXXX） |
| 供应商名称 | 法定公司名称 |
| 纳税人识别号 | 18位统一社会信用代码（核验进项发票时使用） |
| 增值税纳税人类型 | 一般纳税人 或 小规模纳税人（决定能否取得专票） |
| 开票地址及电话 | 发票上的注册地址和联系电话 |
| 开户银行及账号 | 付款时使用 |
| 付款条件 | 约定的付款期限（如：见票即付、月结30天） |
| 应付账款明细科目 | 该供应商对应的总账明细科目（如 2202-V00001） |
| 是否有效 | 是否仍在使用 |

## 27. 销售发票（ARInvoice）

**是什么：** 企业向客户开具的增值税发票，是确认销售收入、形成应收账款和产生销项税额的源头单据。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 发票编号 | 系统内部编号（格式：INV-YYYY-XXXXX） |
| 关联客户 | 购买方客户档案 |
| 发票类型 | 增值税专用发票 / 普通发票 |
| 发票代码及号码 | 税务机关分配的正式发票号码 |
| 开票日期 | 发票开具日期 |
| 到期日 | 按信用期限计算的最晚收款日期（开票日 + 信用期限天数） |
| 不含税金额 | 确认为营业收入的金额 |
| 税额 | 销项增值税额 |
| 含税合计 | 客户应付的全部金额 |
| 核销状态 | 未核销 / 部分核销 / 已核销 |
| 未结余额 | 含税合计 - 已核销金额 |
| 关联日记账凭证 | 对应的收入确认凭证 |
| 关联增值税记录 | 对应的 VATRecord（销项） |

**业务规则：**
- 核销状态随收款核销操作自动更新
- 到期日超过当前日期且未核销的发票计入逾期账款

---

## 28. 采购发票（APInvoice）

**是什么：** 企业从供应商收到的增值税发票，是形成应付账款和确认进项税额的源头单据。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 发票编号 | 系统内部录入编号（格式：PIN-YYYY-XXXXX） |
| 关联供应商 | 销售方供应商档案 |
| 发票类型 | 增值税专用发票 / 普通发票 |
| 供应商发票号码 | 供应商发票上印刷的代码和号码 |
| 收票日期 | 企业收到该发票的日期 |
| 到期日 | 按付款条件计算的最晚付款日期 |
| 不含税金额 | 确认为成本或费用的金额 |
| 税额 | 可抵扣进项增值税额（需满足抵扣条件） |
| 含税合计 | 应付供应商的全部金额 |
| 认证状态 | 待认证 / 已认证 / 认证失败（专票适用） |
| 核销状态 | 未核销 / 部分核销 / 已核销 |
| 未结余额 | 含税合计 - 已付款核销金额 |
| 关联日记账凭证 | 对应的费用或资产入账凭证 |
| 关联增值税记录 | 对应的 VATRecord（进项） |

## 29. 应收核销记录（ARMatching）

**是什么：** 记录每一笔客户收款与具体销售发票之间的匹配关系，追踪每张发票的收款情况。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 关联销售发票 | 被核销的 ARInvoice |
| 关联收款凭证 | 对应的日记账凭证（借：银行存款，贷：应收账款） |
| 核销金额 | 本次核销的金额 |
| 核销日期 | 执行核销操作的日期 |
| 操作人 | 执行核销的财务人员 |

**业务规则：**
- 一张发票可有多条核销记录（分次收款）
- 一张收款凭证可关联多张发票的核销记录（一次付款核销多张）
- 核销金额之和不超过发票含税合计

| 核销日期 | 执行核销操作的日期 |

## 固定资产管理模块实体

## 31. 固定资产卡片（FixedAsset）

**是什么：** 每一件固定资产的完整档案记录，包含资产的基本信息、折旧参数和当前状态。这是固定资产管理的核心实体。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 资产编号 | 系统内唯一标识（格式：FA-XXXXX） |
| 资产名称 | 资产的描述性名称 |
| 资产类别 | 房屋及建筑物 / 机器设备 / 运输设备 / 电子设备 / 其他设备 |
| 取得方式 | 外购 / 自建 / 融资租入 / 非货币性交换 / 债务重组 |
| 取得日期 | 资产正式取得并可使用的日期 |
| 原值 | 资产的初始入账成本 |
| 预计净残值率 | 资产报废时预计残值 ÷ 原值（通常为 3%–5%） |
| 预计净残值 | 原值 × 预计净残值率 |
| 折旧方法 | 直线法 / 双倍余额递减法 / 年数总和法 / 工作量法 |
| 预计使用年限 | 月数（如 60 个月 = 5 年） |
| 使用部门 | 资产所在的业务部门（影响折旧费用科目） |
| 存放地点 | 资产的物理位置 |
| 资产状态 | 正常使用 / 停用 / 已处置 / 已减值 |
| 固定资产明细科目 | 对应总账科目（如 1601-机器设备-某设备） |
| 累计折旧科目 | 对应备抵科目（如 1602-累计折旧） |
| 折旧费用科目 | 折旧计入的损益科目（如 6601-管理费用） |
| 累计已计提折旧 | 系统自动维护的累计折旧总额 |
| 当前账面净值 | 原值 - 累计折旧 - 累计减值损失 |
| 减值准备累计 | 历次计提减值损失合计 |

**业务规则：**
- 资产状态为"停用"时，仍须继续计提折旧
- 资产状态为"已处置"时，停止计提折旧并触发处置流程
- 依据 CAS 4，折旧从取得次月开始计提

## 32. 折旧计划（DepreciationSchedule）

**是什么：** 固定资产按选定折旧方法预先计算出的每月折旧计划，列明整个折旧期内每月应计提的折旧金额。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 关联固定资产 | 所属的 FixedAsset |
| 开始折旧月份 | 取得月份的下一个月 |
| 计划期数 | 总月数（= 预计使用年限月数） |
| 月折旧金额 | 直线法：固定值；加速折旧法：每年重新计算 |
| 折旧计划明细 | 每一期的期初净值、本期折旧额、期末净值 |

**业务规则：**
- 折旧方法或使用年限变更（会计估计变更）须重新生成折旧计划
- 发生减值后，应以减值后账面价值为基础重新计算剩余期间的折旧额

---

## 33. 折旧记录（DepreciationRecord）

**是什么：** 记录每个会计期间实际过账的折旧金额，是已执行折旧操作的历史档案。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 关联固定资产 | 所属的 FixedAsset |
| 折旧期间 | 对应的会计期间（FiscalPeriod） |
| 本期折旧额 | 实际过账的折旧金额 |
| 关联日记账凭证 | 折旧分录对应的凭证 |
| 执行状态 | 已过账 |
| 执行时间 | 折旧凭证的过账时间 |

**业务规则：**
- 每个资产每个期间只能有一条折旧记录
- 折旧记录过账后不可修改，如需更正须通过冲销凭证处理

---

## 34. 资产处置（AssetDisposal）

**是什么：** 记录固定资产报废、出售或其他方式退出使用的处置信息及相关损益。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 关联固定资产 | 被处置的 FixedAsset |
| 处置日期 | 实际处置完成的日期 |
| 处置方式 | 出售 / 报废 / 对外捐赠 / 非货币性交换 |
| 处置收入 | 出售所得价款（报废时为残料变现价值） |
| 处置时账面净值 | 处置日的原值 - 累计折旧 - 累计减值 |
| 处置损益 | 处置收入 - 账面净值 - 处置相关税费（正数为收益，负数为损失） |
| 关联日记账凭证 | 处置分录对应的凭证 |

**业务规则：**
- 处置损益计入"资产处置收益"（6131）或"营业外支出"（6301）科目
- 处置完成后，资产状态变更为"已处置"，停止一切计提

---

## 35. 资产减值（AssetImpairment）

**是什么：** 记录固定资产减值测试的结果和计提的减值损失，依据 CAS 8《资产减值》进行会计处理。

**包含的业务信息：**

| 信息项 | 说明 |
|--------|------|
| 关联固定资产 | 发生减值的 FixedAsset |
| 减值日期 | 确认减值损失的日期（通常为年末） |
| 减值前账面净值 | 减值测试时的账面价值 |
| 可收回金额 | max（公允价值减处置费用, 未来现金流量现值） |
| 减值损失金额 | 账面净值 - 可收回金额 |
| 减值后账面净值 | 可收回金额（减值后的新基础） |
| 关联日记账凭证 | 减值分录（借：资产减值损失，贷：固定资产减值准备） |
| 减值原因说明 | 导致减值的业务背景描述 |

**业务规则：**
- 依据 CAS 8，固定资产减值损失**一旦确认，不可在后续期间转回**
- 减值确认后，须以减值后账面净值重新计算剩余使用年限内的月折旧额
- 减值迹象包括：市价大幅下跌、技术革新导致资产陈旧、经营亏损持续、资产遭受损毁等
